## 1. 启动步骤（必须严格按3步）
### 步骤1：启动游戏（两种方式，推荐方式①）
```
方式① CMD双击： d:\learnning\game1\cmd\启动游戏.cmd
方式② PowerShell： & "D:\tools\game\Godot_v4.6.2\Godot_v4.6.2-stable_win64.exe" --path "d:\learnning\game1"
```

### 步骤2：进入V0.3f演示
在主菜单，点击从左到右数的第**9个橙色按钮**：
```
🛡0.3e樵夫 | ⚔0.3f敌人
          ↑点这个
```
→ 屏幕进入V0.3f敌人AI演示场景。

### 步骤3：操作键位（7键）
| 键 | 作用 | 对应V0.3f演示 |
|---|---|---|
| **A / D** 或 ← → | 玩家左右移动 | 走近/远离敌人触发AI变化 |
| **Space** | 跳跃 | 跳过敌人头顶绕后触发背刺红字 |
| **J** | 普攻斩（J连击可3连） | 打敌人，敌人头顶出伤害浮字白/黄 |
| **K**（长按） | 举盾格挡（扣耐力） | 格挡住敌人攻击→伤害减半白字 |
| **Shift** | 冲刺DASH（消耗耐力） | 快速撤退看敌人RETREAT紫态 |
| **Esc** | 返回主菜单 | 去试其他版本V0.3a~V0.3f |

## 2. 4态AI详细操作验证（QA验收4步）
### 验收① 🟢巡逻PATROL（绿色）
```
操作：一进场景立刻不动，站在x≈300；敌人出生x≈820（距离520>追击150）
观察：敌人会在 740~900 之间左右缓慢走动（±80巡逻）
UI：右上角色卡变【绿】，文字=PATROL
判定：✅通过（3秒内没追你就是对的）
```

### 验收② 🟠追击CHASE（橙色）
```
操作：按D键慢慢向右走，走到 x≈680（敌人x=820，距离=140<150触发线）
观察：敌人停止巡逻，朝玩家快速移动（追过来）
UI：右上角色卡变【橙】，文字=CHASE
判定：✅通过（敌人朝玩家走且速度变快）
```

### 验收③ 🔴攻击ATTACK（红色）
```
操作：继续走到敌人攻击圈58像素内（几乎贴脸）
观察：敌人停步→挥绿色拳头（Slime有攻击弧线）→玩家HP扣减+闪红
UI：右上角色卡变【红】，文字=ATTACK；敌人挥完进入1.1s冷却（橙→红循环）
判定：✅通过（玩家HP减少，敌人挥拳弧线可见）
```

### 验收④ 🟣归位RETREAT（紫色）
```
操作：按Shift冲刺一路向左回到屏幕最左（x≈200）→敌人距离超360
观察：敌人停止追击→转身朝出生点(820)移动
UI：右上角色卡变【紫】，文字=RETREAT
判定：✅通过（敌人回到820后重新进入🟢巡逻）
```

## 3. 伤害浮字3色验证（QA验收3步）
### 浮字颜色规则
| 场景 | 浮字颜色 | 文字内容 | 触发方式 |
|---|---|---|---|
| 普通受击 | ⚪白 | `-18` | 正面J攻击敌人（默认） |
| 暴击 | 🟡黄 | `-暴击58` | 运气暴击（CDC 15%率），多打几次就出 |
| 背刺 | 🔴红 | `背刺36` | 从敌人背后攻击（绕到敌人左侧面向左，敌人朝右=背向） |

### 背刺快速验证法
```
Step1：按D走到敌人右侧（x≈880）
Step2：转身←按A键走到敌人左后方，此时敌人朝向玩家（右）
Step3：快速Space跳过敌人头顶落到敌人背后侧（敌人前方和玩家相反）
Step4：空中按J攻击 → 触发背刺 → 红色"背刺36"浮字！
```

## 4. 失败排查（遇到问题按此表）
| 现象 | 根因（最可能） | 解决 |
|---|---|---|
| 主菜单按钮9点了没反应 | V03f_EnemyDemo.tscn未放对位置 | 检查 scenes/test/V03f_EnemyDemo.tscn 是否存在 |
| 敌人不会动 | SlimeEnemy.gd 未继承EnemyBase | 看类定义 `extends EnemyBase` 是否首行 |
| 受击无浮字 | GameEvents.damage_taken信号没连 | 打开V03f_EnemyDemo.gd _ready确认有ge.connect |
| 玩家被敌人秒 | SlimeEnemy atk设置太高 | config JSON base_atk 改为 8~10（正常） |
| HP条不动 | ProgressBar.value 未在_process中update | Demo脚本 `_process(delta)` 中重赋值 |
