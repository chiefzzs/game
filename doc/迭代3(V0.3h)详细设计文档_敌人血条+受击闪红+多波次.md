## 一、为什么做 V0.3h（必要性）

V0.3f 已经实现了：
1. 敌人 AI 四态 FSM（PATROL/CHASE/ATTACK/RETREAT）
2. 玩家打敌人时显示 3 色伤害浮字（普通白 / 暴击黄 / 背刺红）
3. SlimeEnemy 史莱姆具象化敌人

但玩家还抱怨：
- **看不到敌人还有多少血**：我打了多少？还剩多少？要不要继续追？
- **"打没打中"不确定**：伤害浮字有时被挡住，需要一个更明显的受击反馈
- **一直杀一个史莱姆太无聊**：能不能有递进的战斗节奏，一波打完再来一波更难的？

所以 V0.3h 补这 3 块：
| # | 缺失感 | 对应 V0.3h 功能 |
|---|--------|----------------|
| H1 | 敌人还有多少血？ | **敌人头顶 HP 条**（3 段颜色：绿/黄/红，头顶跟随，掉血渐变） |
| H2 | 打没打中？ | **受击闪红**（被击中那一瞬间敌人通体闪红 → 白闪 0.08s → 恢复） |
| H3 | 战斗没节奏？ | **多波次（3波）WaveManager**：波1(1只史莱姆) → 波2(2只) → 波3(3只)；每波有"第 X 波 / 共 3 波"大字幕提示；全部打完弹出 VICTORY 卡片 |

## 二、用户肉眼可感知的 5 步验收（用户手册 §三 对应）

进入 V03h_WaveDemo 场景后：

1. **看到"第 1 波 / 共 3 波"红色大字幕**（出现 ~1.6s 自动淡出），场景里刷 1 只绿史莱姆
2. **走近史莱姆头顶出现绿色血条**，用 J 攻击它 → 每打一下：敌人闪红 + HP 条掉血（满→空），同时伤害浮字（白/黄/红）
3. **史莱姆死后 2s，"第 2 波 / 共 3 波"蓝色字幕**，场景刷 2 只史莱姆
4. **2 只全部清完 2s，"第 3 波 / 共 3 波"紫色字幕**，场景刷 3 只史莱姆
5. **3 只全清** → 屏幕正中弹出 🏆 **VICTORY 胜利卡片**（波次统计 + 总击杀数 + 耗时 + Esc 回菜单）

### 6.2 WaveManager Autoload API（全新）

```gdscript
## autoload/WaveManager.gd — V0.3h 新增
## ⚠️ 不加 class_name WaveManager（避免和 Autoload 名冲突）

signal wave_started(wave_idx: int, total_waves: int, enemy_count: int)
signal wave_cleared(wave_idx: int, kills_in_wave: int, total_kills: int)
signal all_waves_cleared(total_kills: int, total_seconds: float)

const DEFAULT_WAVES: Array[Dictionary] = [
    {"enemies": 1, "gap_sec": 2.0, "hint": "热身：1 只史莱姆"},
    {"enemies": 2, "gap_sec": 2.2, "hint": "进阶：2 只史莱姆围攻"},
    {"enemies": 3, "gap_sec": 2.4, "hint": "终局：3 只史莱姆围殴"},
]

var waves_cfg: Array[Dictionary] = DEFAULT_WAVES.duplicate()
var current_wave_idx: int = 0         # 0-based, 外显 = idx+1
var enemies_left_this_wave: int = 0
var total_kills: int = 0
var wave_started_at: float = 0.0
var total_elapsed: float = 0.0
var state: int = 0  # 0=IDLE 1=SPAWNING 2=ACTIVE 3=VICTORY

func start_from_first() -> void          # 重开：清零 → 开始第1波
func force_next_wave() -> int            # 演示/调试：跳到下一波
func notify_enemy_killed() -> void       # 接 GameEvents.enemy_died → 计数
```

### 6.3 V03h_WaveDemo 波次大字（UI）

- **wave_started(idx,total,n)** 后：显示 `第 {idx+1} 波 / 共 {total} 波` 屏幕正中大字（48px，白描边红），1.6s 自动淡出
- **wave_cleared** 后：显示 `✔ 第 X 波清完！击杀 N 只` 绿色小字，飘到右上角自动淡出
- **all_waves_cleared** 后：弹出居中 🏆 **VICTORY 卡片**（280×260 白底金边框，3 行统计，Esc 回菜单按钮）

## 九、Godot_v4.6 代码开发必须遵守的原则（追加执行检查）

本迭代严格执行：
- ✅ extends 永远第一行；所有 var/func 参数/返回值全类型
- ✅ Autoload 单例 **不写 class_name**，用 get_tree().root.get_node("X")/has_node 判空
- ✅ `move_and_slide` 只在 `_physics_process` 中调；`_draw` 只画不改状态
- ✅ 信号连接判重（`if not X.is_connected(Y)`）
- ✅ 静态方法创建自身：预加载 `_SELF_SCRIPT := preload(self_path)` 不用 class_name
- ✅ 所有 `get_tree() / autoload / 节点引用` 先判空再 `.`
- ✅ 修改父类字段**先搜全项目**，避免重复声明冲突（如 state_timer 冲突前车之鉴）
- ✅ 所有 .cmd 批处理：CRLF 行尾 + GBK 编码（避免 Terminal 乱码 / "找不到路径"）
