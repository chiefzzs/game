## ① 启动指南（对应 cmd/v0.3 三个脚本，双击即可，无需命令行）

| 脚本名 | 用途 | 启动前必看 |
|--------|------|-----------|
| `cmd/v0.3/01-启动编辑器.cmd` | 启动 Godot 4.6.2 编辑器（地图编辑器 + 战斗场景调试） | 首次使用需**右键编辑本 .cmd**，把第 3 行 `GODOT=...` 改成你本地 Godot 4.6.2 绝对路径；编码必须 GBK+CRLF（乱码则用记事本另存为 → ANSI） |
| `cmd/v0.3/02-启动游戏.cmd` | 直接启动游戏主菜单（含 V0.3 战斗玩法入口） | 同上；若报「系统找不到指定的路径」= 编码不是 GBK，重存即可（Terminal#1-35 根因） |
| `cmd/v0.3/03-自动化验收V0.3.cmd` | 无头模式自动跑 V0.3 全量验收（V0.3a 为坑位版，仅 Phase1-4 + 信号/JSON probe；V0.3i 升级为 6 Phase） | 需提前关闭 Godot 编辑器（否则头less模式会报端口占用）；exit 0 = 全过 |

### ②-1 伤害计算规则表（7 步可视化，V0.3b CombatDamageCalculator 实现）

| 伤害流程 7 步 | 玩家体验感知（屏幕反馈） | 计算规则（CombatDamageCalculator） |
|--------------|------------------------|----------------------------------|
| ① 原始伤害 × 倍率 | 技能段数越高 → 数字越大 | 第 1 段 100% · 第 2 段 110% · 第 3 段 130%；箭伤害 × 0.9 |
| ② 防御减伤 | 重甲敌人 → 白字变小 | 最终伤害 = max(0, 原始伤害 − 防御力 × 1.0) |
| ③ 格挡 / 破盾 | 盾满 → 蓝字（伤害 − 80%）+ 体力条扣；体力空 → 红盾+破盾音效+伤害 × 2.0 | 正面 ±60° 且按 K；破盾武器（樵夫盾斧）强制破盾 |
| ④ 背刺 | 背后击中 → 红数字 2×，字体放大 22 | 敌人背面 ±60° 判定；背刺必暴击（guarantees_crit=true） |
| ⑤ 暴击 | 黄数字 1.75× + 粒子闪光 | 默认 5%；猎户 + 15%（弓专精）；背面额外 + 20% |
| ⑥ 钳制（保底 / 上限） | 新手 1 级砍 BOSS 永远 ≥ 1 伤；满血永远不会被一击秒 | 下限 1 点 / 上限 = 目标最大 HP × 100%（玩家保护） |
| ⑦ 击退向量 | 击中后敌人后仰 + 轻微扬飞，第 3 段连击击退最大 | 玩家被打 180 速 / 敌人被打 120 速 + y = −60（上扬 60px） |

## ⑦ 常见问题与排查（坑位版，Terminal#历史错误汇总）

| 错误现象 | 99% 根因 | 修复步骤（30 秒解决） |
|---------|---------|---------------------|
| 双击 .cmd → 「系统找不到指定的路径」+ 中文全是乱码 | O05 违反：.cmd 用 UTF-8 保存了（GBK 铁律） | ① 右键 .cmd → 记事本打开 → 文件 → 另存为 → 编码选「ANSI（= GBK）」→ 保存覆盖 → 再双击<br>② VSCode 打开 .cmd → 右下角点 UTF-8 → 通过编码保存 → 简体中文 GB2312 |
| 启动游戏 → 黑框秒退 / 弹 Parse Error: Unexpected 'const' | G01 违反：某脚本（大概率稻草人 Scarecrow.gd）把 const 写在 extends 前面了 | 看报错栈最后一行 .gd 路径 → 打开该脚本 → 第一行必须是 `extends Xxx`，第二行 `class_name`，第三行才准写 const/var |
| 启动无报错但稻草人 AI 一动不动 / 找不到 Player | O03 违反：CharacterBase.take_damage() 函数签名被改了（V0.2 稻草人用旧签名）→ 连带初始化炸了 → 回归 cmd/v0.2/03-automation，exit code 变了 | Git 回退 scripts/editor/CharacterBase.gd 到 V0.2 基线，按 O03：**新增 take_damage_ex()，旧 take_damage(int) 一字不改保留** |
| 伤害有时 0 / 有时 99999（数值全乱） | A03 违反：某角色脚本直接 `FileAccess.open("player.json")` 读，没走 ConfigManager.cfg_get → L1/L2 merge 的增量值读不到 | 全局替换所有 `FileAccess.open("res://config/...json` → 全改成 `Config.GetL2("player.baseHp", 100)` |
| 无头验收（Step 0 / cmd 03）→ 全 Pass，但手动玩 AI 全程发呆 | T02 违反：AI 视野/寻敌逻辑写在 `_draw()` 里了（无头模式 _draw 永不调用） | 把 AI 决策代码全挪到 `_physics_process(delta)`；`_draw()` 只允许画 HP 条/视野圈这种纯渲染代码 |
| 测试时「连击数翻倍 / 金币翻倍」一次加 N 份 | D01 违反：`GameEvents.x.connect(...)` 热重载后连了 N 次（没判 is_connected） | 所有 connect 代码统一套 D01 模板：`if ge and not ge.xxx.is_connected(_cb): ge.xxx.connect(_cb)` |
