## 一、技术选型与整体架构

### 1.1 技术选型建议

| 类别 | 推荐方案 | 备选方案 | 说明 |
|------|----------|----------|------|
| **游戏引擎** | Godot 4.x | Unity 2D / Cocos2d-x | 开源免费、2D功能完善、自带TileMap地图编辑支持 |
| **编程语言** | GDScript | C# / C++ | 与Godot深度集成，开发效率高 |
| **地图编辑** | Godot TileMap + 自定义编辑器插件 | Tiled Map Editor | 引擎原生支持，减少数据转换成本 |
| **数据格式** | JSON | YAML / TOML | 通用易读，编辑器友好 |
| **美术资源** | Aseprite / LibreSprite | Photoshop | 像素风格绘制工具 |
| **音频** | WAV(音效) + OGG(BGM) | MP3 | 无损音效 + 压缩音乐 |

### 1.2 整体架构分层（MVC变体）

```

│                     表现层 (View Layer)                       │

                              │ 事件/信号
                              ▼

│  - 角色控制器    - 战斗管理器    - 招募系统    - 经济系统     │
│  - 敌人AI        - 状态机        - 任务系统    - 存档系统     │
└──────────────────────────────────────────────────────────────┘
                              │ 数据读写
```

## 二、核心模块详细设计

### 2.1 角色模块（Character Module）

#### 2.1.1 基类设计（CharacterBase）

```
CharacterBase (Node2D / KinematicBody2D)
├── 属性
│   + String  characterId          // 角色唯一ID
│   + int     maxHp                // 最大血量
│   + int     currentHp            // 当前血量
│   + int     attack               // 攻击力
│   + int     defense              // 防御力
│   + float   moveSpeed            // 移动速度
│   + float   jumpForce            // 跳跃力
│   + bool    isAlive              // 是否存活
│   + Facing  facingDir            // 朝向 (LEFT/RIGHT)
│
├── 引用
│   + Sprite2D       sprite
│   + CollisionShape2D  hitBox     // 受击碰撞盒
│   + Area2D         attackBox     // 攻击判定区域
│   + AnimationPlayer animPlayer
│
└── 方法
    + void TakeDamage(int damage)  // 受伤
    + void Heal(int amount)        // 治疗
    + void Die()                   // 死亡处理
    + void Move(Vector2 dir)       // 移动
    + void Attack()                // 攻击
    + signal HpChanged(int newHp)
    + signal CharacterDied()
```

```
PlayerController extends CharacterBase

│   + CompanionManager companionMgr
```

### 2.2 战斗系统（Combat System）

#### 2.2.1 伤害计算器（DamageCalculator）

```
DamageCalculator (Singleton)
└── 方法
    + int CalculateDamage(CharacterBase attacker,
                          CharacterBase defender,
                          bool isBlocked = false)
    │
    │   算法:
    │   baseDamage = attacker.attack - defender.defense * 0.5
    │   baseDamage = max(1, baseDamage)
    │   if isBlocked: baseDamage *= 0.1   // 格挡减伤90%
    │   return round(baseDamage)
    │
    + void ApplyDamage(CharacterBase attacker, CharacterBase defender)
```

#### 2.2.2 命中盒系统（HitBoxSystem）

```
攻击判定规则:
- 攻击时激活AttackBox (Area2D)
- AttackBox.body_entered信号触发伤害判定
- 同一攻击动作内同一目标只判定一次（用HashSet记录已命中目标）
- 格挡判定: defender.isBlocking AND 朝向正对攻击者
```

#### 2.2.3 战斗状态机

```
PlayerState (枚举)
  ├── IDLE           // 待机
  ├── WALKING        // 移动
  ├── JUMPING        // 跳跃中
  ├── FALLING        // 下落中
  ├── DASHING        // 冲刺中（无敌帧）
  ├── BLOCKING       // 格挡中
  ├── ATTACKING_1    // 一段攻击
  ├── ATTACKING_2    // 二段攻击
  ├── ATTACKING_3    // 三段攻击
  ├── HURT           // 受击硬直
  └── DEAD           // 死亡

状态转换约束:
- ATTACKING_* → 不能切换到 WALKING/JUMPING 直到动作完成
- BLOCKING → 只能移动？不能，格挡时禁止移动
- DASHING → 期间忽略其他输入（除打断）
- HURT → 有0.2~0.4秒硬直，期间无法操作
```

## 三、输入系统设计

### 3.1 按键映射表（Input Map）

| 动作名称 (Action) | Xbox手柄 | 键盘(默认) | 说明 |
|-------------------|----------|------------|------|
| `move_left`       | L摇杆←   | A / ←      | 左移 |
| `move_right`      | L摇杆→   | D / →      | 右移 |
| `jump`            | A键      | 空格 / W   | 跳跃（时长控制高度） |
| `block`           | LT键     | 左Shift    | 格挡（按住） |
| `dash`            | Y键      | K / LShift | 冲刺 |
| `attack`          | X键      | J          | 攻击 |
| `interact`        | B键      | E          | 交互（对话/宝箱/商店） |
| `pause`           | Menu键   | Esc        | 暂停 |

### 3.2 输入处理流程

```
每帧流程:
1. InputManager 收集原始输入
2. 转换为Action级输入（带缓冲）
3. PlayerController 判断当前状态是否允许接受输入
4. 执行对应动作 → 切换状态机
```

### 3.3 输入缓冲（Input Buffer）

```
跳跃缓冲 (Coyote Time + Jump Buffer):
- Coyote Time: 离开平台后0.1秒内仍可跳跃（容错）
- Jump Buffer: 落地前0.12秒内按跳跃，落地瞬间自动起跳
- 提升操作手感，降低精确帧要求
```

### 4.1 编辑器整体架构

```
地图编辑器采用"引擎内置插件 + 独立EXE分发"双模式方案（★用户早期参与核心设计）：

│                   地图编辑器主界面                            │
├───────────────┬───────────────────────┬─────────────────────┤
│  图块选择面板   │     画布编辑区         │   属性/层级面板     │
│  (Tile Palette)│   (Canvas + Grid)     │  (Inspector)        │
│               │                       │                     │
│ □地面图块     │  ┌─────────────────┐  │ - 当前选中图块属性  │
│ □平台图块     │  │  1:1 预览编辑    │  │ - 实体参数编辑     │
│ □装饰图块     │  │  网格吸附模式    │  │ - 触发器配置      │
│ □碰撞图层     │  │  缩放/平移      │  │ - 图层可见性      │
│               │  └─────────────────┘  │ - 地图元数据       │
│ [图块集选择器] │                       │ - ⚙️ 目标配置面板   │
├───────────────┴───────────────────────┴─────────────────────┤
│  工具栏: [新建] [打开] [保存] [撤销] [重做] [测试] [导出JSON]│
│  启动入口:  [🗺️  新建关卡 ]  [📂 打开用户目录]  [🎮 一键测试]│
└─────────────────────────────────────────────────────────────┘
```

#### 4.1.1 用户双模式启动（★用户早期参与核心）
编辑器必须支持**两种启动方式**，保证非开发用户也能使用：

| 启动模式 | 适用用户 | 启动方式 | 说明 |
|----------|---------|----------|------|
| **A. 独立EXE模式** | 玩家/社区/普通用户 | 单独的 `MapEditor.exe` | 无需安装引擎，开箱即用；**V0.2 即打包对外分发** |
| **B. 游戏内置入口** | 玩家 | 游戏主菜单「地图工坊」→「编辑关卡」 | 无需切换程序，保存后直接从「用户关卡」游玩 |

用户创作数据互通约定：
- 所有用户地图统一保存到：`%APPDATA%/Game1/UserMaps/`（Windows）或 `~/Game1/UserMaps/`
- 文件格式：`.map.json`（双后缀，方便引擎识别与用户备份）
- 内置5个官方模板：`tpl_empty.map.json` / `tpl_farm.map.json` / `tpl_battle_arena.map.json` 等

### 4.2 地图分层设计（Layer Stack）

每张地图采用**多层TileMap**结构，从上到下顺序渲染：

| 图层ID | 图层名 | 用途 | 参与碰撞 | 说明 |
|--------|--------|------|----------|------|
| 7 | `foreground` | 前景装饰（遮罩） | ❌ | 遮挡角色，增加纵深感 |
| 6 | `entity_top` | 实体上层（头顶UI等） | ❌ | |
| 5 | `entities` | 实体层（NPC/敌人/宝箱） | 特殊 | 对象放置层 |
| 4 | `triggers` | 触发器层 | Area2D | 检查点、事件区、传送门 |
| 3 | `platforms_oneway` | 单向平台 | ✅ 仅上方 | 可从下跳上的平台 |
| 2 | `decorations` | 场景装饰（花草/箱子） | ❌ | 不阻挡 |
| 1 | `ground_solid` | 实心地面/墙壁 | ✅ 全方向 | 核心碰撞层 |
| 0 | `background` | 背景远景/贴图 | ❌ | 视差滚动层 |

### 4.4 地图编辑器核心功能需求

#### 4.4.1 基础编辑功能
- **新建地图**：设置尺寸（宽×高 图块数）、图块大小（默认32×32）
- **打开/保存**：读取/写入 `.json` 地图文件
- **撤销/重做**：编辑历史栈（建议≥50步）
- **图块刷**：左键放置、右键删除、按住拖动连续绘制
- **填充工具**：矩形/自由区域填充同一图块
- **选择工具**：框选批量移动/删除/复制粘贴
- **橡皮擦**：擦除指定图层图块
- **网格吸附**：可选开关，默认开启（对齐32px网格）
- **缩放平移**：鼠标滚轮缩放（0.5x~4x）、中键拖动平移画布

#### 4.4.2 图层管理功能
- **图层列表**：显示/隐藏/锁定/重排 图层
- **活动图层高亮**：仅可编辑当前选中图层
- **碰撞可视化**：
  - 实心地面 → 绿色边框
  - 单向平台 → 蓝色边框（上方）
  - 实体/触发器 → 黄色/红色可编辑矩形

#### 4.4.3 实体放置功能
- **实体库面板**：按分类列出可放置实体
  - NPC类：可招募NPC、商人、剧情NPC
  - 敌人类：普通兵、精英、弓箭手、骑士
  - 互动物：宝箱、门、传送门、梯子、商店
  - 拾取物：金币、武器、道具
- **放置方式**：从实体库拖入画布 → 自动对齐网格
- **参数编辑**：选中实体后在属性面板修改（巡逻范围、宝箱内容、目标地图等）

#### 4.4.4 触发器编辑功能
- **触发器绘制**：在画布上拖拽出矩形区域
- **事件绑定**：从预设事件列表中选择
  - `SHOW_HINT` 显示提示
  - `SAVE_CHECKPOINT` 保存检查点
  - `SPAWN_ENEMY` 刷怪
  - `TRANSITION_MAP` 切图
  - `TRIGGER_DIALOG` 触发对话
  - `START_CINEMATIC` 播放过场
- **触发条件**：fireOnce（一次性）、requirePlayer（仅玩家触发）、enable（开关）

#### 4.4.5 测试与导出功能
- **一键测试**：保存地图 → 启动游戏直接进入该地图（玩家出生在spawnPoint）
- **JSON校验**：导出前校验必填字段、引用资源是否存在
- **导出JSON**：输出符合Schema的 `.json` 文件到 `res://data/maps/` 目录
- **预览渲染**：编辑区实时显示效果，与游戏内运行一致

## 五、关卡流程管理系统设计（★新增P0模块）

### 5.1 层级结构与生命周期

```
全局生命周期:
  GameManager._Ready()
    → ChapterFlowManager.LoadChapterConfig("chapters.json")
    → ProgressFlags.LoadFromSave(如果有存档) or 新游戏初始化
    → 进入第一个章节的第一张地图
      → MapLoader.Load(mapId)
        → 创建 LevelFlowController 节点并挂载到 MapRoot
        → LevelFlowController.Init(objectives[], triggers[], introCin)
          → 进入 LevelState.INTRO (开场动画/对话)
          → 动画结束 → LevelState.PLAYING
            → 监测目标完成/失败条件
          → 目标完成 → LevelState.OBJECTIVE_COMPLETE
            → 写入 ProgressFlags "objective:{objId}=true"
            → 如有 outroCinematic → 播放
            → ChapterFlowManager 判定章节解锁
            → SceneManager 切换到下一张地图
```

### 5.2 LevelFlowController（关卡内流程）类设计

```
LevelFlowController extends Node
├── 属性
│   + String mapId
│   + LevelState state = LOADING
│   + bool paused = false
│   + Array<Objective> objectives            // 从map.json加载
│   + Dictionary<String, bool> objCompleted  // 运行中状态
│   + LevelFlowControllerConfig config       // 时间阈值等
│
├── 引用
│   + Node mapRoot
│   + ChapterFlowManager chapterMgr
│   + ProgressFlags flags
│
└── 方法
    + void Init(Dictionary meta, Array objectives, Array triggers)
    + void TransitionTo(LevelState newState)
    + void EvaluateObjectives()          // 每帧调用，评估目标
    + bool IsAllRequiredObjectivesDone() -> bool
    + void TriggerObjectiveReward(String objId)
    + void FailLevel(String reason)      // 失败流程（重试/读档菜单）
    + void CompleteLevel()               // 完成流程（切图）
    + signal LevelStateChanged(oldState, newState)
    + signal ObjectiveCompleted(objId, reward)
    + signal LevelFailed(reason)
    + signal LevelCompleted(nextMapId)
```

#### 5.2.1 状态转换表

| 当前状态 | 事件 | 下一状态 | 动作 |
|----------|------|----------|------|
| LOADING | Map加载完毕 | INTRO | 播放introCin/弹出「XX关卡」提示 |
| INTRO | 过场结束/用户跳过 | PLAYING | 释放玩家控制 |
| PLAYING | 用户按暂停键 | PAUSED | 显示暂停UI + 暂停游戏进程 |
| PAUSED | 用户取消暂停 | PLAYING | 恢复 |
| PLAYING | 所有 required目标完成 | OBJECTIVE_COMPLETE | 奖励结算+触发outroCin |
| PLAYING | 玩家HP==0 | FAIL_STATE | 死亡UI + 回检查点选项 |
| OBJECTIVE_COMPLETE | outro结束/跳过 | EXITING | 调用SceneManager切到nextMap |
| EXITING | 新地图加载完毕 | 销毁本实例 | 内存回收 |

#### 5.2.2 目标系统计算器（ObjectiveEvaluator）

```
内置目标类型处理器字典:
{
  REACH_AREA:       (cfg) => 玩家是否进入 triggerCfg.area,
  DEFEAT_ENEMY:     (cfg) => flags.Has("defeated:{cfg.instanceId}"),
  DEFEAT_ALL_IN_MAP:(cfg) => 本图enemies全部死亡,
  RECRUIT_NPC:      (cfg) => flags.Has("recruited:{cfg.npcDataId}"),
  OPEN_CHEST:       (cfg) => flags.Has("opened:{cfg.chestInstanceId}"),
  TRIGGER_FIRED:    (cfg) => flags.Has("triggered:{cfg.triggerId}"),
  DIALOG_COMPLETE:  (cfg) => flags.Has("dialog:{cfg.dialogId}"),
  BOSS_DEFEATED:    (cfg) => flags.Has("boss_defeated:{cfg.bossId}"),
  AND:              (cfg) => cfg.subs.All(IsCompleted),
  OR:               (cfg) => cfg.subs.Any(IsCompleted),
  NOT:              (cfg) => !IsCompleted(cfg.sub)
}
```

## 十一、性能优化策略

### 11.1 对象池（Object Pooling）

```
适用对象:
- 敌人（离屏或死亡后回池）
- 特效/粒子（打击特效、受击数字）
- 投射物（箭矢）
```

### 11.2 视锥剔除

```
- TileMap 使用 Y-Sort + 视口剔除
- 超出摄像机范围±2屏的敌人：暂停AI更新
- 远处同伴保持跟随但不做复杂索敌
```

### 11.3 分块加载（大型地图）

```
超大地图按 20x20 图块分块 (Chunk):
- 进入视野的Chunk加载Tile
- 离开视野的Chunk卸载（保留实体状态）
```
