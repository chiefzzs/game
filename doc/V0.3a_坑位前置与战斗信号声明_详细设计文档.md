## 三、cmd/v0.3 目录规范（A7，对应 §6 One Track C2 + O05 GBK 编码）

> **为什么必须 cmd/v0.3 独立目录？** 防止 `replace_all("V0.2", "V0.3")` 误伤 cmd/v0.2 历史验收脚本（C2 约束永不覆盖旧版本）。

### 3.1 3 个脚本的替换规则（V0.2 → V0.3 复制，全局正则替换）：

| .cmd 文件 | 原 V0.2 字符串 | 替换后 V0.3 字符串（全局替换） |
|-----------|--------------|----------------------------|
| 01-启动编辑器.cmd | `V0.2` → `V0.3`；`01-LaunchEditor` → `01-V03-LaunchEditor`；标题行「Map Editor Mode」→「Map Editor + Combat Test Mode」 |
| 02-启动游戏.cmd | `V0.2` → `V0.3`；`02-LaunchGame` → `02-V03-LaunchGame`；「Workshop button」→「Workshop + Combat Mode Entry」 |
| 03-自动化验收V0.3.cmd | `V0.2` → `V0.3`；`03-AutoAcceptV0.2` → `03-V03-AutoAccept`；Phase 标题追加「坑位阶段 + V0.3 signal/JSON probe」；Phase1 文件检查新增 5 JSON + CharacterEnums + GameEvents 信号探针 |

### 3.2 GBK + CRLF 编码强制（O05 铁律，否则 Terminal#1-35 重现）：

- 所有 .cmd 保存时 **严禁 UTF-8 / UTF-8 BOM**，必须用「ANSI = GBK」
- 行尾必须 `\r\n`（CRLF），Git 检出自动转换配置已在 .gitattributes（如有）
- 验证编码：PowerShell `[System.IO.File]::ReadAllBytes("xxx.cmd") | Select-Object -First 3` → 不含 EF BB BF（UTF-8 BOM 标记）= ✅ 合规
