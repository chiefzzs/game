## §2 CharacterBase FSM 设计（严格状态跳转表）

### §2.1 FSM 状态枚举（**追加到 V0.2 CharacterKind 枚举之后，旧值一字不动**）
```gdscript
enum FSMState {   # V0.3c 新增：8 核心状态 + 2 扩展（连击 2/3 段）
	IDLE,         # 0：静止 / 待机
	RUN,          # 1：左右移动（V0.2 原 move 逻辑映射）
	JUMP,         # 2：跳跃上升/下落（含 double jump）
	ATTACK1,      # 3：一段攻击（J 键短按）
	ATTACK2,      # 4：二段连击（V0.3d 玩家实装；V0.3c 仅保留跳转位）
	ATTACK3,      # 5：三段连击（V0.3d 扩展位）
	HURT,         # 6：受伤硬直（take_damage 自动切入；hitstun 到 0 自动回到 IDLE）
	BLOCK,        # 7：举盾格挡（按住 LT/L2 切入；松开回 IDLE）
	DEAD          # 8：死亡（HP<=0 自动切入；禁止切出任何状态）
}
```

### §2.2 状态跳转规则（change_state(to) 前置检查，非崩溃 = 返回 Godot Error 枚举）
| From 状态 | 允许 → To 状态 | 禁止（Godot Error 值） |
|-----------|----------------|----------------------|
| IDLE      | RUN / JUMP / ATTACK1 / HURT / BLOCK / DEAD | ATTACK2/3（ERR_DOES_NOT_EXIST = 14） |
| RUN       | IDLE / JUMP / ATTACK1 / HURT / BLOCK / DEAD | ATTACK2/3（无 combo 基础） |
| JUMP      | IDLE / RUN / ATTACK1 / HURT / DEAD | BLOCK（空中不可格挡） |
| ATTACK1   | ATTACK2 / HURT / DEAD | IDLE/RUN（攻击动画锁死，提前切会断招） |
| ATTACK2   | ATTACK3 / HURT / DEAD | IDLE/RUN（同上） |
| ATTACK3   | IDLE / HURT / DEAD | RUN/JUMP（三段结束后才可移动） |
| HURT      | IDLE / DEAD | 任意战斗状态（硬直时间到才准恢复） |
| BLOCK     | IDLE / HURT / DEAD | ATTACK*（举盾中不可攻击） |
| DEAD      | 无 | 全部（ERR_DOES_NOT_EXIST） |

### §2.3 change_state 接口实现伪代码
```gdscript
func change_state(to: int) -> Error:
	if state == FSMState.DEAD and to != FSMState.DEAD:  # DEAD 绝对不可出
		return ERR_DOES_NOT_EXIST
	if not _legal_transition.has(state):  # 兜底非法 state 值
		return ERR_INVALID_PARAMETER
	var allows: Array[int] = _legal_transition[state]
	if to not in allows:
		return ERR_DOES_NOT_EXIST  # 不允许 → state 保持原样返回错误码（非崩溃）
	prev_state = state
	state = to
	_on_state_enter(prev_state, to)  # 子类可 hook
	return OK
```

### §2.4 take_damage 接入 CDC 的 **V0.2 兼容方案**（核心 One Track 保证）
- **旧签名 100% 冻结**：`func take_damage(dmg: int, source: Node = null, opts: Dictionary = {}) -> int`（参数顺序/类型/默认值 — **与 V0.2 字节级相同**，外部子类调用老代码零修改）
- **CDC 调用条件（仅当 opts 同时满足以下 4 键时启用，否则走 V0.2 老逻辑）**：
  1. `opts.has("_use_cdc")` 为 `true`（显式开关，默认关 → V0.2 默认走老路）
  2. `opts.has("attacker_dict")`（Dictionary，攻击者 stat，见 V0.3b 10UC）
  3. `opts.has("victim_override")` 可为空 Dictionary，覆盖 CDC 读 self 的 stat（方便单元测试）
  4. `opts.has("context_dict")`（DamageType / attack_angle / base_damage 等）
- **CDC 接入流程（开启时）**：
  1. 预取 CDC 脚本：`const _CDC_SCRIPT := preload("res://scripts/combat/CombatDamageCalculator.gd")`（V0.3a 测试已确保脚本存在；首帧 preload 失败 fallback 老逻辑）
  2. 构造 victim stats：`self` 的 hp/max_hp/def/… + `victim_override` 覆盖（测试专用）
  3. 调用 `cdc.calculate_damage(attacker_dict, victim_stats, context_dict)` → 取返回字典 r
  4. 扣血：`set_hp(hp - r.final_damage, source)`；切 FSM：`if state != DEAD: change_state(FSMState.HURT)`
  5. 应用击退：`velocity.x += sign(r.knockback.x) * abs(r.knockback.x)`（避免死循环重复击退）
  6. 发 V0.2 旧信号 `hit.emit(r.final_damage, source, r.is_crit)`（**旧代码订阅方零修改**）
  7. 返回 `r.final_damage`（返回值类型仍为 int，兼容 V0.2）
- **未满足 CDC 开启条件** → 直接执行 V0.2 原 take_damage 所有行（DUMMY/巡逻兵等 V0.2 角色 100% 行为不变）

### §2.5 安全 _autoload 工具函数（杜绝 Engine.get_singleton，审计检查项）
```gdscript
func _autoload(name: String) -> Node:
	if get_tree() == null or get_tree().root == null:
		return null
	return get_tree().root.get_node_or_null("/root/" + name)
```
→ 所有需要 GameEvents / Config / InputBus 的位置统一用 `_autoload("GameEvents")`，无空指针崩溃。

## §5 风险与回滚方案（One Track 安全兜底）

### §5.1 风险 1：V0.2 老角色 take_damage 行为变了？
- **规避方案**：CharacterBase 单元测试 T11~T14（私有，不入库，验收时本地执行）= 对 Dummy/Patrol 旧代码调用 `take_damage(5, null, {})`，opts 不含 `_use_cdc` → 走 V0.2 原逻辑，最终 HP 计算结果与 V0.2 baseline 字节级一致
- **回滚**：如出现异常 → 只需把 CharacterBase.gd 的 FSM 追加段注释掉（120~220 行），其他文件全不碰即可回退到 V0.3b 状态

### §5.2 风险 2：FSM ATTACK 状态锁导致 V0.2 move 不可用？
- **规避方案**：V0.2 子类没有调用 change_state()，FSM state 字段恒为 0=IDLE；V0.2 move_and_slide 逻辑不检查 state → 行为零变化；状态锁仅在显式调用 change_state 后启用
- **回滚**：改 change_state 的 `allows` 数组，对 IDLE 的 To 状态追加全量（暂时取消锁）

### §5.3 风险 3：preload CDC 脚本不存在导致 CharacterBase 无法加载？
- **规避方案**：take_damage CDC 路径开头加 `if not _CDC_SCRIPT: return _old_take_damage_branch(...)`，同时 V0.3a SmokeTest 已加断言 preload CDC 存在性
