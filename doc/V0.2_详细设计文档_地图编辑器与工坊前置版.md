### 模块 A：CharacterBase 基类（T02-P01 并行前置）

**文件**：`scripts/editor/CharacterBase.gd`  
**设计目的**：V0.3 玩家/同伴/敌人开发前置，统一所有角色的基础行为与信号接口，避免重复代码。

#### 2.1 导出属性 & 成员变量

| 属性 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `max_hp` | `int` | `100` | 最大生命 |
| `stamina` | `float` | `100.0` | 体力（格挡/冲刺消耗） |
| `max_stamina` | `float` | `100.0` | 最大体力 |
| `facing` | `int` | `1` | 朝向：1=右 / -1=左 |
| `hp` | `int` | 内部（`set_hp` 封装） | 当前生命（初始化=max_hp） |
| `is_dead` | `bool` | `false` | 死亡标志（死亡后 take_damage 直接返回0） |
| `is_invulnerable` | `bool` | `false` | 无敌（复活/受击帧无敌） |
| `hitstun` | `float` | `0.0` | 硬直计时（=0 时角色可操作） |
| `DEFAULT_HITSTUN` | `float` | `0.15` | 默认硬直秒数 |
| `DEFAULT_KNOCKBACK` | `float` | `120.0` | 默认击退像素/秒 |
| `velocity` | `Vector2` | `Vector2.ZERO` | 兼容 CharacterBody2D 语义（后续 Player 继承可直接 move_and_slide） |

#### 2.2 核心方法

| 方法签名 | 返回 | 说明 |
|---|---|---|
| `take_damage(dmg:int, source:Node=null, opts:Dictionary={}) -> int` | `实际扣血` | 无敌/死亡则返回0；否则计算最终扣血=clamp(round(dmg),1,hp)，应用硬直/击退，触发 `set_hp`，0 HP 触发 `die()` |
| `heal(amount:int) -> int` | `实际恢复量` | 最大恢复到 max_hp，返回实际增加的 HP |
| `set_facing(direction:int) -> void` | — | 规范化 direction∈{-1,1}，变化时发射 `face_changed` |
| `set_invulnerable(enabled:bool, duration:float=0.0) -> void` | — | 永久切换 / duration>0 时 `create_timer(duration).timeout` 自动关闭 |
| `set_hp(new_hp:int, killer:Node=null) -> void` | — | clamp到[0,max_hp]，旧值不同则发 `hp_changed`；==0 且未死 → `die(killer)` |
| `die(killer:Node=null) -> void` | — | 标 is_dead=true，发 `died(killer)` 信号 |

#### 2.3 信号（4个对外总线）

| 信号 | 参数 | 触发时机 |
|---|---|---|
| `hp_changed(old_hp:int, new_hp:int, max_hp:int)` | — | HP 被 set_hp 修改且旧≠新 |
| `hit(damage:int, source:Node, final_pos:Vector2)` | — | take_damage 成功扣血 |
| `died(killer:Node)` | — | 首次从 hp>0 到 hp==0（只发一次） |
| `face_changed(new_facing:int)` | — | 朝向切换 |

#### 2.4 与 V0.3 继承关系（预设计）

```
CharacterBase.gd (extends Node2D，提供 HP/信号/基础API)
   ├── PlayerBase     (extends CharacterBody2D，组合 CharacterBase 作为成员或再继承)
   │     └── FarmerPlayer (真正 V0.3 玩家脚本)
   ├── EnemyBase      (extends CharacterBody2D)
   │     ├── ScarecrowEnemy / SoldierEnemy / KnightBossEnemy
   │     └── ArcherEnemy
   └── NpcBase        (extends CharacterBody2D)
         ├── FarmerNPC / ArcherNPC / WarriorNPC
```

### 模块 B：LevelFlowController 关卡流程状态机（T02-P03 前置）

**文件**：`autoload/LevelFlowController.gd`  
**注册名**：`LevelFlow`（在 `project.godot` 的 `[autoload]` 段注册为 Singleton）  
**设计目的**：统一 V0.3+ 所有关卡的状态流转（开始/暂停/目标/失败/下一关），并被 MapLoader 自动注入 objectives。

#### 2.5 7 种状态枚举

```gdscript
enum State {
    LOADING,      # 0：加载关卡（MapLoader 写入 Layers/Entities）
    INTRO,        # 1：开场动画/淡入/显示「农场 - 新手村」Title（默认3秒）
    PLAYING,      # 2：玩家可操作，目标可完成
    PAUSED,       # 3：暂停（Esc/Start 切进切出）
    OBJECTIVE_COMPLETE,  # 4：所有目标完成，显示「关卡通过」结算面板
    FAIL_STATE,   # 5：失败（HP=0/掉坑等），提供 Retry / Exit
    EXITING       # 6：退出过渡（淡出 0.5s 后 change_scene）
}
```

#### 2.6 12 个对外 API（面向 GameMain / PauseMenu / FailPanel / ObjectivePanel）

| API 签名 | 作用 |
|---|---|
| `start_level(level_id:String, objectives:Array=[], next_id:String="", intro_sec:float=3.0)` | 设置当前关卡元信息 + 初始化 objectives 池 → 置 LOADING → 0.1s 后切 INTRO → intro_sec 后切 PLAYING（自动倒计时） |
| `get_state() -> State` | 返回当前状态（GameMain._process 里据此判断是否接受输入） |
| `pause_game() -> bool` | PLAYING→PAUSED，true=成功（非 PLAYING 返回 false 不切换） |
| `resume_game() -> bool` | PAUSED→PLAYING |
| `toggle_pause() -> bool` | 自动在 PLAYING/PAUSED 间切（Esc 直接绑这个） |
| `complete_objective(obj_id:String) -> Dictionary` | 标记 objective={id,status=done,reward}；若 objectives 全部 DONE → 切 OBJECTIVE_COMPLETE 并发 all_objectives_completed；返回该 objective 原始 dict（null 表示未找到） |
| `is_objective_done(obj_id:String) -> bool` | UI 查询用（显示✅勾） |
| `get_objectives_status() -> Array[Dictionary]` | ObjectivePanel 直接显示：每项 {id,type,status,data,reward,label} |
| `trigger_checkpoint(cp_id:String, pos:Vector2, flags:Dictionary={})` | 记录 checkpoint 位置 + 标记 flags（例 restore_hp=true → 再死 retry 回满血） |
| `trigger_fail(reason:String) -> void` | 切 FAIL_STATE，发 level_failed |
| `retry_level() -> void` | FAIL → 触发 state_exited(FAIL_STATE) 后 GameMain 自行 reload 当前场景；用 checkpoint 恢复 |
| `exit_level(next_id:String="") -> void` | 切 EXITING；若 next_id 空则用 start_level 时传的 next_id；0.5s 发 level_exit(next) 让 GameMain 调 change_scene |
| `exit_to_menu() -> void` | 退出到主菜单（封装 exit_level→"res://scenes/main_menu/MainMenu.tscn"） |

#### 2.7 8 个信号

```gdscript
signal state_entered(new_state: State)
signal state_exited(old_state: State)
signal objective_completed(obj_id: String, reward: Dictionary, first_time: bool)
signal all_objectives_completed(objectives: Array)   # 切到 OBJECTIVE_COMPLETE 前发
signal checkpoint_triggered(cp_id: String, pos: Vector2, flags: Dictionary)
signal level_failed(reason: String)
signal level_exit(next_level_id: String, next_scene_path: String)
signal intro_started(level_id: String, intro_sec: float)
```

#### 2.8 状态机严格转换规则（非法转换返回 false 或忽略）

```
LOADING ─(加载OK 0.1s)─▶ INTRO ─(intro_sec超时)─▶ PLAYING ─┬─(Esc/Start)─▶ PAUSED
                                                           ├─(all_objectives_done)─▶ OBJECTIVE_COMPLETE ─(点下一关)─▶ EXITING
                                                           └─(player die)─▶ FAIL_STATE ─(Retry)─▶ LOADING(retry)
PAUSED ─(Esc)─▶ PLAYING                         OBJECTIVE_COMPLETE / FAIL_STATE ─(点回主菜单)─▶ EXITING
EXITING ─(0.5s淡出)─▶ change_scene()
```

#### 2.10 EditorMain.gd 对外核心逻辑

1. **_ready()**：连接 7 按钮 + TilePalette/EntityPalette 信号 + Inspector 信号 + ObjectivesEditor 信号；默认模式 MODE_DRAW(0) + 图层 L2(2)
2. **_unhandled_input(event)**：
   - `Ctrl+S` → `_on_save_pressed()`
   - `Ctrl+Z` → `draw_tools.undo()`，同步刷新画布
   - `Ctrl+Y / Ctrl+Shift+Z` → `draw_tools.redo()`
   - `B/E/F` → 笔刷/橡皮/矩形填充
   - 在 CanvasViewport 区域：`_handle_canvas_input(event)`（区分左键/右键/中键/拖动）
3. **_handle_canvas_input**：
   - MODE_DRAW 下：左键拖动画笔（B）/ 矩形拖选填充（F）/ 右键擦除（E）；每步 `draw_tools.paint_cell / flood_fill_area` 同步 TileMap.set_cell
   - MODE_ENTITY 下：左键放置当前选中的 entity_kind → 在 EntitiesRoot 创建 Node2D（颜色/图标区分6类）并设 meta(id,kind,props)；右键/中键点中一个实体 → 切换为 MODE_EDIT，EntityInspector 自动选中并渲染字段
4. **7 按钮逻辑**：
   - **新建**：MapSerializer.build_map_dict 新建空 → `_load_map_dict_to_canvas` 清 8 层 + 清空 EntitiesRoot + 默认 REACH_AREA 目标
   - **编辑已有**：FileDialog 选 .map.json → MapSerializer.load_from_file → Validator.validate → 报错显示状态栏，否则 _load_map_dict_to_canvas
   - **官方模板**：PopupMenu 3项 empty/farm/arena → 从 `res://scenes/workshop/templates/xxx.map.json` 读入（自动调用 `_ensure_builtin_templates`，不存在就 WorkshopMain 规则创建兜底写回 templates 目录，然后再读）
   - **一键测试**：先 MapSerializer.save_to_file 到 `user://tmp_editor_test.map.json`；然后 `OS.create_process([GODOT路径, "--path", 项目根绝对, "--" 或 Flags 约定 next_run_load_map=该路径])` → 启动第二个 Godot 进程并在玩家进入游戏后由 GameMain 检测到 Flag，调 MapLoader.load_map_to_parent 直接进关卡（GameMain V0.3骨架中预留该判断，V0.2 已把 Flag 的定义写入 WorkshopMain 说明）
   - **保存 / 另存为**：先 `MapSerializer.build_map_dict(layers_8 从 TileMap.get_used_cells_by_id 遍历、entities 从 EntitiesRoot.get_children 读 meta、objectives_editor.to_json_array、meta.spawn = PlayerSpawnMarker.position)` → `MapSchemaValidator.validate` → 若 errors≠0：状态栏标红 + 拒绝保存 + `LblErrors.text = validator.errors_to_string()`；通过再 write file。

#### 2.11 画布坐标系约定（后续版本严格遵守，避免坐标坑）

- TileMap `tile_size = Vector2i(40, 40)`（V0.2 常量值，写在 MapSerializer 的 BUILD_TILE 常量里）
- 画布内部分辨率固定：1920×1080 → 横向最多 48 tile（1920/40），纵向最多 27 tile（1080/40）
- 实体 meta.x / meta.y 用 **像素坐标（即 Node2D.position 整数）**，**不用 tile 坐标**
- PlayerSpawn.position 同样是像素坐标，默认值 `Vector2i(200, 800)`（L2地面 tile_y=14 * 40 = 560？不对，地面y=14 是 tile 行=14，角色脚站在 y=15*40=600，所以 spawn_y=560-48=512；V0.2官方模板默认 spawn 均按此换算过的像素坐标写入 `meta.spawn`）

### 模块 D：DrawTools 绘制工具 + 50步撤销栈

**文件**：`scripts/editor/DrawTools.gd`（extends RefCounted，纯逻辑类）  
**设计目的**：EditorMain.gd 创建 `var draw_tools := DrawTools.new()`，所有对 TileMap 的修改都经过 DrawTools 封装，这样撤销/重做可完整恢复。

#### 2.12 工具模式 & 撤销数据结构

```gdscript
enum Tool { BRUSH, ERASER, RECT_FILL }
const MAX_UNDO := 50

# 单个 Undo 项：修改的 (layer, Vector2i coords) -> AtlasCoords old/new 全量记录
# 用 Dictionary key = "%d,%d" % [layer, x, y]，value = { "old": PackedInt32Array([altas_id,altas_x,altas_y]), "new": same }
# 这样 undo = 循环写回 old 值；redo = 循环写回 new 值；不会因不同工具重复实现逻辑。
```

#### 2.13 API

| API | 说明 |
|---|---|
| `set_tool(t: Tool) / get_tool() -> Tool` | 切换 B/E/F |
| `set_layer(l: int) / get_layer() -> int` | 切换当前操作图层 0-7（只在当前层写） |
| `begin_action(label: String="") -> void` | 开始一段原子操作（例：鼠标按下开始拖动画笔） |
| `end_action() -> void` | 结束原子操作，推入 undo_stack（若 change_dict 非空）；如果栈长>MAX_UNDO pop_front |
| `paint_cell(tilemap: TileMap, layer: int, coord: Vector2i, new_atlas: PackedInt32Array) -> void` | 写入一个 cell：先从 tilemap.get_cell_atlas_coords 读 old，记录 change_dict；再 tilemap.set_cell(layer, coord, src_id=0, new_atlas) |
| `erase_cell(tilemap: TileMap, layer: int, coord: Vector2i) -> void` | 本质 paint_cell(new_atlas=空 PackedInt32Array，等价于 TileMap.set_cell(..., -1)) |
| `flood_fill_area(tilemap: TileMap, layer:int, c0:Vector2i, c1:Vector2i, pattern: PackedInt32Array) -> int` | 矩形填充：把包含 c0/c1 的 AABB 的每个坐标都调 paint_cell；返回填充 cell 数 |
| `bfs_fill(tilemap: TileMap, layer:int, seed:Vector2i, pattern: PackedInt32Array, max_cells:int=2000) -> int` | 洪水填充（TilePalette 预留「油漆桶」按钮 V0.3 实装） |
| `undo(tilemap: TileMap) -> bool` | undo_stack 出栈 -> 把每个 key 的 old 写回 TileMap；该项移入 redo_stack |
| `redo(tilemap: TileMap) -> bool` | redo_stack 出栈 -> 把每个 key 的 new 写回 TileMap；该项移入 undo_stack |
| `undo_available() / redo_available() -> bool` | 工具栏按钮灰化（V0.2 预留，V0.3 可接 ToolTip 显示剩余步数） |

#### 2.18 顶层逻辑 & 10 种目标处理器类型

```
顶层（objectives[0]）： { logic: "AND" | "OR" }
   含义：AND = objectives[1..] 全部 DONE 才通关（默认）
         OR  = objectives[1..] 任一 DONE 即通关
```

| type（枚举） | 参数 data（必须项） | reward（选填） | 何时 DONE（V0.3 由游戏逻辑调用 complete_objective） |
|---|---|---|---|
| `REACH_AREA` | {x,y,w,h} 像素矩形 | gold/exp/unlock | 玩家 global_position 进入该矩形（Trigger trigger_reach_area action=objective_done） |
| `DEFEAT_ENEMY` | {enemy_ids: [id1, id2]} | gold/exp/unlock | 所有 enemy_ids 的 CharacterBase.died 均触发 |
| `RECRUIT` | {npc_ids: [npc1]} | npc_ids 加入同伴组 | 支付 recruit_cost 且 NPC 接受招募后 |
| `OPEN_CHEST` | {chest_id} | 直接=chest loot | 玩家按 E 开宝箱成功时 |
| `TRIGGER` | {trigger_id: String} | gold/unlock | Trigger 子类型 action=objective_done(obj_id=this_trigger_id) |
| `DIALOG` | {dialog_id} | none | 对话系统 DialogManager.play(dialog_id) 全部 finish |
| `BOSS` | {enemy_id: boss_id} | BOSS专属奖励（title_xxx） | boss.enemy died + 满足 is_boss=true |
| `AND` / `OR` | {sub_ids:[obj_id_a,obj_id_b]}（组合目标） | 子目标各自奖励 | 子 AND/OR 全部/任一完成（V0.3 ObjectivePanel 递归判断） |
| `NOT` | {obj_id} | none（惩罚/限制） | 原 obj 没被完成（例："不能碰陷阱"） |

#### 2.20 程序校验规则（和下面 JSON Schema 保持双向一致）

| 检查点 | path | 失败条件 |
|---|---|---|
| 顶层必填 7 字段 | `.` | 缺任一：version/map_id/display_name/tilesets/layers/entities/objectives |
| version 格式 | `$.version` | 非字符串或不以 "0." 开头（V0.x 系列） |
| tilesets 非空 + 每项 id/name/first_id 整型/字符串/整型 | `$.tilesets[*]` | 不符合即报错 |
| layers 必须 8 项 + 严格按 index=0..7 顺序，每层 `cells` 必须是 Array 且子项是 length=3 的整数数组 `[x,y,atlas_packed_int]` | `$.layers[index] / $.layers[index].cells[i]` | 数量≠8 / 任何项缺 index 缺 name / cells[i] 不是 [int,int,int] |
| entities 每项必填 4 字段：`id(unique字符串) / kind(6种枚举) / x:int / y:int / props: dict` | `$.entities[*]` | kind 不在6种内 / id 重复 / x 或 y 不是 int / props 不是 dict |
| objectives[0] 必须是 `{logic}`：AND/OR；objectives[1..] 每项必填 `id+type+data:dict`；id 唯一 | `$.objectives[*]` | 缺 [0] / id 重复 / type 不在 10 种处理器类型枚举 |
| meta.spawn 若存在必须是 `{x:int, y:int}` | `$.meta.spawn` | spawn 不是 dict / x非int |

3. **Step 3 Objectives + LevelFlow 注入**：若 `use_autoload=true`，则：
   ```gdscript
   LevelFlow.start_level(
       map_dict.get("map_id", "custom"),
       map_dict.objectives.slice(1),             # 第0项是 logic，V0.3 ObjectivePanel 递归实现，start_level 先收 objectives[1..]
       map_dict.get("meta", {}).get("next_level", ""),
       3.0
   )
   ```
   → LevelFlow FSM 自动：LOADING → INTRO → PLAYING
4. **Step 4 返回 summary**：记录 issues（例：`kind=enemy 模板=boss，V0.2 当前使用占位 Enemy Node2D，V0.3 将加载真实 EnemyBase.tscn`）。
