## 二、本次迭代交付的功能清单（迭代2 = V0.2 全部模块）

### 🧱 模块 A：CharacterBase 基类（并行开发组 T02-P01）
- 位置：`scripts/editor/CharacterBase.gd`
- 统一角色基础属性：`hp/max_hp/atk/facing/hitstun/is_invulnerable`
- 统一接口：`take_damage() / heal() / set_facing() / set_invulnerable() / die()`
- 信号：`hp_changed / hit / died / face_changed`
- 玩家、同伴（NPC）、敌人都可以继承此基类以避免重复代码。

### 🎯 模块 B：LevelFlow 关卡流程单例（并行开发组 T02-P03）
- 位置：`autoload/LevelFlowController.gd`，注册名 `LevelFlow`
- 7 状态 FSM：`LOADING → INTRO → PLAYING ⇄ PAUSED → OBJECTIVE_COMPLETE → EXITING → (下一关)`  以及 失败 `FAIL_STATE`
- 对外 API：
  - `start_level(level_id, objectives=[], next="", intro=3.0)`
  - `pause_game() / resume_game() / toggle_pause()`
  - `complete_objective(obj_id)`
  - `trigger_checkpoint(cp_id, pos, flags)`
  - `trigger_fail(reason) / retry_level() / exit_to_menu() / exit_level(next_id)`
- 8 个信号：`state_entered / state_exited / objective_completed / all_objectives_completed / level_failed / level_exit`

### 🛠 模块 C：独立地图编辑器主体
- 场景：`scenes/editor/EditorMain.tscn`   脚本：`scripts/editor/EditorMain.gd`
- 三栏布局：
  - **顶部工具栏 7 按钮**：📄新建 / 📂编辑已有 / 📚官方模板 / 🎮一键测试 / 💾保存Ctrl+S / 📋另存为
  - **左侧 Tab（面板 ×2）**：🗂 TilePalette 图块面板  /  📦 EntityPalette 实体面板
  - **中间 1920×1080 画布**：`CanvasRoot → TileMap(8层) + EntitiesRoot(Node2D) + PlayerSpawnMarker`
  - **右侧 Tab（面板 ×2）**：🔍 Inspector 实体属性编辑 / 🎯 ObjectivesEditor 目标配置
  - **底部状态栏**：模式 / 图层 / 文件 / 状态信息（4 个 Label）

### 🖌 模块 D：DrawTools 绘制工具
- 位置：`scripts/editor/DrawTools.gd`
- 工具：笔刷 / 橡皮 / 矩形填充（TilePalette 面板+画布切换）+ BFS 洪水填充接口
- 撤销栈：`MAX_UNDO = 50`，支持 `undo()` / `redo()`
- 快捷键：`B=笔刷 E=橡皮 F=矩形填充  Ctrl+Z撤销  Ctrl+Y重做  Ctrl+S保存  Esc取消`

### 🧱 模块 E：TilePalette 图块面板 + 5 套官方图块集
- 位置：`scripts/editor/TilePalette.gd`（已挂 EditorMain.tscn 左 Tab TilePalette 节点）
- 8 层图层：L0远景 / L1近景 / L2地面⭐ / L3装饰 / L4前景 / L5特效 / L6碰撞 / L7标记
- 5 套 ID 区间（无纹理版本用不同颜色占位，保持 ID 语义稳定）：
  - 🌾 Farm: 1~99   🏘 Village: 100~199   🌲 Forest:201~299   🏰 Castle:301~399   ⚖ Admin:401~499
- 共 23 个可用 tile 按钮 + 8 个图层按钮

### 📦 模块 F：EntityPalette 实体面板（6 大类 20 个官方模板）
| 类别       | 模板实体数 | 典型内容                                                                    |
| ---------- | ---------- | --------------------------------------------------------------------------- |
| 🤝 NPC     | 3          | Farmer John / Archer Lisa / Warrior Tom（可招募，含 recruit_cost 属性）     |
| 👹 Enemy   | 4          | Scarecrow 稻草人 / Soldier 士兵 / Archer 弓手 / Knight 骑士                |
| 📦 Chest   | 3          | Gold / Potion / Mixed(锁)                                                   |
| 🚩 CheckPoint | 1       | 默认检查点，cp_id 自定义                                                    |
| 🚪 Portal  | 2          | →Next / ↩Back                                                               |
| ⚡ Trigger | 4          | ReachArea / Dialog / SpawnWave / CameraZone                                 |

- **放置方式**：选中模板 → 在画布左键点击；右键/中键选中一个实体后可在右 Inspector 改属性。

### 🔍 模块 G：EntityInspector 实体属性编辑
- 位置：`scripts/editor/EntityInspector.gd`
- 按 6 种 kind 自动推荐字段（hp/atk/patrol_r/chase_r/attack_r/cd/loot/target_level 等 35+ 字段）
- 信号：`entity_updated(path, field, value) / entity_removed(path)`

### 🎯 模块 H：ObjectivesEditor 目标配置（10 种处理器 + AND/OR 组合）
- 位置：`scripts/editor/ObjectivesEditor.gd`
- 组合逻辑：顶层 `AND`（全完成）/ `OR`（任一完成）
- 10 种目标处理器类型：
  `REACH_AREA / DEFEAT_ENEMY / RECRUIT / OPEN_CHEST / TRIGGER / DIALOG / BOSS / AND / OR / NOT`
- API：`add_objective() / remove_objective() / update_objective() / load_objectives() / to_json_array()`

### ✔ 模块 I：MapSchemaValidator + map.schema.json
- **程序校验器**：`scripts/editor/MapSchemaValidator.gd`
  - 顶层 7 个必填字段 / layers 固定 8 项 / entities 6 字段 / objectives id+type 必填
  - `validate(dict) -> Array[{path, message}]` / `is_valid() -> bool` / `errors_to_string()`
- **官方 JSON Schema**（Draft 2020-12）：`config/schemas/map.schema.json`
  - 可在 VSCode/VSCodium 中用 `"$schema": "./config/schemas/map.schema.json"` 获得 JSON 自动补全与错误波浪线。

### 💾 模块 J：MapSerializer 保存 / 加载
- 位置：`scripts/editor/MapSerializer.gd`
- 写盘文件格式：`.map.json`（统一 JSON），并自动生成同名 `.bak` 备份。
- 用户地图目录：`%APPDATA%/Godot/app_userdata/MedievalRebellion/UserMaps/`，由 `user_maps_dir()` 创建。
- API：`build_map_dict() / save_to_file() / load_from_file() / resolve_map_path(filename)`

### 🧩 模块 K：MapLoader 游戏端加载
- 位置：`scripts/editor/MapLoader.gd`
- 调用方式：`load_map_to_parent(parent:Node2D, map_data:Dictionary, use_autoload=true)`
- 完成：8 层 TileMap 写入 cells / 6 类实体实例化并挂载 metadata / objectives 自动注入 `LevelFlow.start_level`
- 返回：`{layers_instantiated, entities_instantiated, cells_count, error_count, issues}`

### 🏭 模块 L：游戏内「⚒ 地图工坊」UI
- 场景：`scenes/workshop/WorkshopMain.tscn`   脚本：`scripts/editor/WorkshopMain.gd`
- 主界面 4 大按钮：
  - 📄 **新建空地图** → 打开 EditorMain.tscn（新建空图状态）
  - 📂 **编辑已有地图** → 扫描 UserMaps 目录，打开第一张（或选中后加载）
  - 📚 **官方模板** → 拷贝模板 → 进入编辑器
  - 🎮 **测试用户地图** → 加载 .map.json 进入 GameMain
- 自动兜底：若官方模板缺失，启动时会一键生成 3 张（无需手动放文件）。

### 📚 模块 M：3 张官方地图模板

## 三、地图编辑器使用速查（90 秒上手）

1. **启动编辑器**：双击 `cmd\v0.2\01-启动编辑器.cmd`，Godot Editor 打开本工程
2. **进入编辑器运行态**：
   - 方式 A（推荐）：在 FileSystem 双击 `scenes/editor/EditorMain.tscn`，按 `F6` 以该场景运行
   - 方式 B：按 `F5` 从主菜单 → 选 **⚒ 地图工坊** → 📚官方模板 / 📄新建
3. **画地面**：左 Tab「TilePalette」选中 `⭐ L2 平台/地面` + 点绿草色 Tile → 中画布左键拖动画一条地面
4. **放实体**：左 Tab 切「EntityPalette」→ 选稻草人 🎃 → 画布左键放置
5. **改实体属性**：右键/中键画布上的稻草人 → 右 Inspector 中自动切换为 enemy 的推荐字段（hp/atk/patrol_r/cd ...）
6. **写目标**：右 Tab「ObjectivesEditor」→ 逻辑 AND（✅ 全部完成）→ `+击杀` 按钮 → 目标 ID 填 `sc_die_1`
7. **保存**：Ctrl+S（另存为会自动写到用户 UserMaps 目录）
8. **一键测试**：点顶部「🎮 一键测试」→ 自动保存临时文件 → OS.create_process 启动第二个 Godot 实例加载该地图

## 七、与主菜单 / 游戏流程的集成点

- `MainMenu` 入口：主菜单第二级按钮「⚒ 地图工坊」→ `get_tree().change_scene_to_file("res://scenes/workshop/WorkshopMain.tscn")`

## 八、常见问题 FAQ

1. **批处理提示找不到 Godot.exe？**
   编辑批处理首行 `GODOT=...` 改为你本机 Godot 4.6.2 的绝对路径即可。

2. **「编辑已有地图」说用户目录没有地图？**
   先「📚官方模板」或「📄新建空地图」→ 用编辑器 Ctrl+S 保存一次，地图就会写入 `%APPDATA%/Godot/app_userdata/MedievalRebellion/UserMaps/`，再点刷新即可看到。

3. **保存时被校验拒绝？**
   看编辑器底部状态栏红色错误信息：`必填字段缺失 / cells 不是数组 / entity缺少props` 等，按提示补齐后再保存。

4. **如何把地图发给好友？**
   直接把 UserMaps 里的 `.map.json` 发给好友，让他放进同一路径即可（注意不需要额外素材：tile ID 由本手册模块E 约定统一）。
