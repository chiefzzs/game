## 二、7 步伤害流水线（严格按顺序，每步输入=上步输出）

> 算法公共方法签名：`func calculate_damage(attacker_stats: Dictionary, victim_stats: Dictionary, context: Dictionary) -> Dictionary`
> - **attacker_stats**（必填）：攻击方属性字典，键见 §2.1
> - **victim_stats**（必填）：被击方属性字典
> - **context**（必填）：攻击上下文（距离、角度、格挡标签……），见 §2.2
> - **返回值**：7 步计算后的详情 Dictionary，键名 100% 对齐 GameEvents.damage_calculated 第 3 参数 details（V0.3h HUD 零转换直接用）

### 2.1 输入键规范（3 类字典的必选/可选键）

| 类别 | 必选键（无则断言异常 T03 仅测试脚本允许） | 可选键（无则用 combat_formula 默认） |
|------|------------------------------------------|--------------------------------------|
| attacker_stats | `atk: int`（基础攻击力）、`facing: int`（±1，CharacterEnums.Facing） | `crit_rate_bonus: float=0`、`is_shield_break: bool=false`、`kind: int=ENEMY`（CharacterKind） |
| victim_stats | `def: int`（防御力）、`hp: int`、`hp_max: int`、`position: Vector2`、`facing: int` | `stamina: int=0`、`stamina_max: int=0`、`is_blocking: bool=false` |
| context | `base_damage: int`（技能原始伤害，无则=attacker_stats.atk） | `attack_angle_rad: float=NaN`（击打点与受害者朝向夹角，用于背刺判定）、`hit_position: Vector2=victim pos`、`damage_type: int=PHYSICAL`（DamageType 枚举） |

### 2.2 7 步执行顺序 + 输出键（combat_formula.json 对应键完全一致）

```
第 1 步：原始伤害 → 伤害类型倍率修正
    raw_1 = max(1, context.base_damage ? context.base_damage : attacker_stats.atk)
    dt_mult = formula.damage_types[DamageType.keys[context.damage_type]] ? formula.damage_types[...] : 1.0
    dmg = round_to_int(raw_1 * formula.base_damage_mult * dt_mult)

第 2 步：防御减伤（线性减）
    def_effective = victim_stats.def * formula.defense_subtract
    dmg = max(formula.defense_min_clamp, dmg - def_effective)

第 3 步：格挡判定（正面 + victim.is_blocking=true）
    if context.attack_angle 在 victim.facing 正面 ± formula.block.angle_deg / 2 且 victim.is_blocking:
        blocked = true
        stamina_cost = round_to_int(dmg * (1 - formula.block.default_reduction) * formula.block.stamina_per_damage)
        if victim.stamina < stamina_cost:  # 体力不足 → 破盾
            shield_broken = true
            dmg = round_to_int(dmg * formula.block.shield_break_multiplier)
        else:
            dmg = round_to_int(dmg * (1 - formula.block.default_reduction))

第 4 步：背刺判定（attacker 在 victim 背面 ± formula.backstab.angle_deg / 2）
    if formula.backstab.enabled and context.attack_angle 判定为背面:
        is_backstab = true
        dmg = round_to_int(dmg * formula.backstab.damage_multiplier)
        if formula.backstab.guarantees_crit:  # 背刺必暴击
            crit_rate_bonus += 1.0

第 5 步：暴击（roll 0~1 < base_rate + 背刺加成 + attacker 加成 → 暴击）
    crit_rate = clamp(formula.crit.default_rate + formula.crit.from_behind_bonus_rate*(背面?1:0) + attacker.crit_rate_bonus, 0, 1)
    if randf() < crit_rate:
        is_crit = true
        dmg = round_to_int(dmg * formula.crit.default_multiplier)

第 6 步：钳制（下限 1 伤害 / 上限 = 目标最大 HP * max_by_target_hp_pct，避免一击秒满血）
    dmg = clamp(dmg, formula.clamps.min_final_damage, round_to_int(victim.hp_max * formula.clamps.max_by_target_hp_pct))
    final_damage = dmg

第 7 步：击退向量（根据 attacker.facing 方向计算 knockback_vector:Vector2）
    kb_x = attacker.facing * (victim.kind == PLAYER ? 180 : 120)
    knockback = Vector2(kb_x, -60)  # 轻微上扬
```

### 2.3 输出 Dictionary（14 字段 = 100% 覆盖 GameEvents.damage_calculated + HUD 浮动文字需求）

```gdscript
return {
    "steps": {
        "s1_raw": int, "s2_after_defense": int, "s3_after_block": int,
        "s4_after_backstab": int, "s5_after_crit": int, "s6_clamped": int
    },
    "final_damage": int,                      # 最终扣血量
    "is_crit": bool, "is_backstab": bool,     # 背刺/暴击标记
    "is_blocked": bool, "shield_broken": bool,# 格挡/破盾标记
    "block_stamina_cost": int,                # 本次消耗体力（盾破=0）
    "knockback": Vector2,                     # 击退向量
    "floating_text_color": String,            # #FFFFFF/#FFD166/#EF476F/#8FD3FF
    "floating_text_font_size": int,           # 22暴击 / 16普通
    "damage_type": int                        # 透传 DamageType
}
```

## 四、CombatDamageCalculator 类设计规范（§7 铁律合规）

| 铁律编号 | 合规说明 |
|---------|---------|
| G01 extends 第一行 | `extends RefCounted` 第一行，第二行注释，第三行开始 const/preload |
| G02 全类型 | `calculate_damage(a: Dictionary, v: Dictionary, ctx: Dictionary) -> Dictionary`，所有局部变量 `var x: int/float/bool/Vector2` 无 untyped |
| G03 class_name 白名单 | **不写 class_name**（L4 算法类，用 preload 引用，避免全局扫描冲突） |
| A01/A02 JSON 规范 | combat_formula.json 只读：`FileAccess.get_file_as_string` + `JSON.parse`，不写回；键用 dot 路径封装 `_get_formula(path: String, default)` 内部函数 |
| A04 无 Autoload | 整个类**零引用 /root/GameEvents、/root/Config**；配置直接读 JSON 文件 → 测试零打桩（承诺一） |
| T01 铁律 | 类本身**无 OS.exit / get_tree().quit**；仅测试脚本 V03b_DamageTest.gd extends SceneTree 才能 quit |
| D02 | 所有 JSON parse 结果判空：`if parse_result.error != OK: push_warning + use hardcoded default`，永不 crash |

## 五、用户手册增量（B3：填充手册 ② 操作表 + ⑤ 拾取物）

### 5.1 手册 §② 玩家操作末尾追加「伤害计算规则表（7 步可视化）」

| 伤害流程 7 步 | 玩家体验感知 |
|--------------|------------|
| ① 原始伤害×倍率 | 技能段数越高 → 数字越大（第 3 段 130% 原始） |
| ② 防御减伤 | 敌人有盾/重甲 → 数字白字变小（需破盾） |
| ③ 格挡/破盾 | 盾满时→蓝字（伤害-80%），体力空→红盾+破盾音效 |
| ④ 背刺 | 玩家绕到敌人背后 → 红数字 2× |
| ⑤ 暴击 | 黄数字 1.75× + 字体放大 22% |
| ⑥ 钳制 | 玩家血条 80% 以下才会被一击秒（上限 100% HP，新手不死保护） |
| ⑦ 击退 | 击中后敌人轻微后仰 + 扬飞，玩家连击更容易命中 |

### 5.2 手册 §⑤ 拾取物末尾追加「战斗后 Roll 表：伤害越高 → 掉落加成」

| 掉落加成规则 | 触发条件 | 效果 |
|-------------|---------|------|
| 击杀伤害≥敌人 50%HP | 玩家/同伴造成 final_damage ≥ enemy.hp_max × 0.5 | roll 表 +10% 大金币权重 |
| 背刺击杀 | is_backstab=true + enemy died | 额外 Roll 1 次小血药 |
| 破盾击杀 | shield_broken=true | 1% 概率掉稀有武器蓝图（V0.4 坑位） |
