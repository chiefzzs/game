## 2. 战斗伤害计算系统 (CombatDamageCalculator)

### 2.1 计算7步流水线

```
 raw (atk × mult × type)
   → ① defense_subtract (max(min_clamp, raw - def×mult))
   → ② front_check (方向)
   → ③ block (减伤%, 消耗stamina)
   → ④ backstab (在背面时 +damage% +guarantee_crit)
   → ⑤ crit (概率暴击 × mult)
   → ⑥ clamp (min/max/不回血)
   → ⑦ knockback
```

### 2.2 10个覆盖用例

| #   | 场景                        | 期望值                                   |
|-----|-----------------------------|------------------------------------------|
| UC1 | 普通攻击 atk=30 def=10      | final_damage 落在 [18,22]                |
| UC2 | 暴击触发                    | is_crit=true, final ≥ base×1.5           |
| UC3 | 格挡                        | is_block=true, final降低, stamina_cost>0 |
| UC4 | 背刺                        | is_backstab=true, 伤害>正面同参数        |
| UC5 | 最小伤害钳制                | final_damage ≥ combat_formula.min_clamp  |
| UC6 | 最大伤害钳制                | final_damage ≤ max_clamp                 |
| UC7 | 死者 miss                   | is_miss=true, final=0                    |
| UC8 | 物理/火属性倍率差异         | 不同type → dmg差≥10%                     |
| UC9 | 被格挡时体力不足仍扣血      | stamina<cost时仍能造成部分伤害           |
| UC10| 盾破武器绕过格挡            | 盾破武器 + 格挡状态 = 不触发is_block     |

### 2.3 输出结果字段

```gdscript
{
  ok, final_damage:int, raw_with_mult:float, reduced_after_def:float,
  is_front:bool, is_block:bool, blocked_damage:float, stamina_cost:float,
  is_backstab:bool, is_crit:bool, is_miss:bool, knockback:float,
  steps:Array[String], reason:String
}
```

## 3. 角色FSM设计

### 3.1 状态机 (BaseState enum)

```gdscript
enum BaseState { IDLE=0, MOVE=1, JUMP=2, ATTACK=3, HURT=4, BLOCK=5, DEAD=6, DASH=7 }
```

| 状态      | 允许进入的下一个状态                       | 核心行为                                    |
|-----------|-------------------------------------------|-------------------------------------------|
| IDLE      | MOVE, JUMP, ATTACK, BLOCK, HURT, DEAD, DASH | 等待输入/AI决策, 恢复stamina              |
| MOVE      | IDLE, JUMP, ATTACK, BLOCK, HURT, DEAD, DASH | 速度=speed, 消耗stamina(跑步)              |
| JUMP      | MOVE, IDLE, ATTACK, HURT, DEAD             | velocity.y=-jump_force, 重力自然下落        |
| ATTACK    | HURT, IDLE, DEAD                           | 动画帧→hitbox激活→调用CombatDamageCalculator |
| HURT      | IDLE, DEAD                                 | 受击硬直 + Knockback, hp减少              |
| BLOCK     | MOVE(慢速), IDLE, HURT, DEAD, ATTACK       | block_reduction生效, 每秒扣stamina         |
| DEAD      | (无)                                      | 碰撞关闭 → 延迟移除/播放死亡动画, 掉落地掉落物 |
| DASH      | IDLE, MOVE, HURT, DEAD                     | 无敌帧 + 高速位移 + 冷却                    |

## 4. CombatHUD 设计

### 4.1 UI节点树

```
CombatHUD (CanvasLayer)
  ├─ TopLeft
  │   ├─ PlayerHPBar          (TextureProgress + label: HP/MaxHP)
  │   ├─ PlayerStaminaBar     (绿色耐力条)
  │   ├─ GoldDisplay          (金币图标 + 动态数字)
  │   └─ WeaponSlots          (3格武器 + 当前选中highlight)
  ├─ TopCenter
  │   └─ ComboCounter         (连击x数 + 淡出计时)
  ├─ TopRight
  │   └─ ObjectiveTracker     (目标列表 + √)
  └─ BottomCenter
      └─ EnemyHPBars          (当前在攻击范围内敌人的血条，动态生成/销毁)
```

### 4.2 事件订阅

```gdscript
GameEvents.character_stats_changed  →  HP/Stamina条更新
GameEvents.gold_changed             →  GoldDisplay刷新
GameEvents.combo_applied            →  ComboCounter x数+重置淡出
GameEvents.objective_status_changed →  ObjectiveTracker更新
GameEvents.enemy_hp_changed         →  EnemyHPBars刷新
```

## 5. 拾取 & 掉落系统

### 5.1 PickupItem.gd (Area2D)

- body_entered 时检测 tag==player → PickupSystemAutoload.collect_item(id, amount) → queue_free()
- 视觉: 1s 浮动动画 + 拾取瞬间 sfx + 闪光粒子

### 5.2 掉落表 (pickups.json → drops)

```json
{
  "items": {
    "gold":      {"type":"currency","value_default":1,"color":"#FFD700","shape":"circle"},
    "potion_hp": {"type":"consumable","value_default":25,"color":"#FF6B6B","shape":"square"},
    "potion_mp": {"type":"consumable","value_default":20,"color":"#4DABF7","shape":"square"}
  },
  "drops": [
    {"enemy_id":"Enemy_WalkSoldier","item_id":"gold","amount_min":5,"amount_max":15,"prob":0.95},
    {"enemy_id":"Enemy_WalkSoldier","item_id":"potion_hp","amount_min":1,"amount_max":1,"prob":0.25},
    {"enemy_id":"Enemy_JumpScout", "item_id":"gold","amount_min":8,"amount_max":20,"prob":1.0},
    {"enemy_id":"Enemy_Dummy",     "item_id":"gold","amount_min":0,"amount_max":0, "prob":0.0}
```

### 5.3 敌人死亡掉落

```gdscript

    PickupSystem.spawn_pickup(global_position, d.item_id, d.amount)
```

## 8. 架构依赖约束 (V0.3 必须遵守)

```
Delivery (cmd/scenes/test)
  ↑
Scenes (characters/**/*.tscn)
  ↑   Scripts (characters/combat/ui/systems)
  ↑      ↑
Autoload (GameEvents/ConfigManager/PickupSystem...)
  ↑
Config (L1/L2/L3 JSON)   Resource (sprites/audio)
```

**绝对禁止反向依赖**：  
- 严禁 Scripts/Autoload 层依赖 Scenes/Delivery 层  
- 严禁 CharacterBase 直接引用具体子类 (`class_name` 仅注册，不反向 preload)  
- 严禁直接 JSON.parse_file() → 统一走 ConfigManager.cfg_get(路径, 默认值)
