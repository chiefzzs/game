### 1.2 关键路径（Critical Path 调整后：把编辑器+用户参与前置）

```
V0.1 框架+ConfigManager
  → V0.2 编辑器(独立EXE)+工坊入口+ProgressFlags  ★用户早期参与窗口
  → V0.3 玩家控制+LevelFlowController
  → V0.4 战斗系统+ChapterFlowManager+招募
  → V0.5 内容实装(15张地图+3BOSS)
  → V0.6 打磨+设置菜单4Tab+用户导入导出
  → V1.0 正式发布
```

**关键路径说明**：V0.2 是最大的前置调整点——**编辑器必须提前交付**，这样用户在V0.3就可以开始提交自己的地图，V0.5开始整合到正式章节里，把社区创作当作免费内容池。

## 二、整体迭代路线图（★对齐新增模块 + 编辑器前置）

| 版本号 | 代号 | 核心目标 | 预计周期 | 关键交付物（★新增模块标注） |
|--------|------|----------|----------|------------------------------|
| **V0.1** | 脚手架+配置骨架版 | 工程框架、核心单例、ConfigManager(L1~L4)、ProgressFlags、基础输入/碰撞、存档壳 | 1周 | 可运行空项目 + 4层配置加载无报错 + 玩家设置可保存/读回 |

| **合计** | - | - | **≈10.5周** | - |

> **路线图核心变化**：
> 1. **V0.2 扩容到2周（最长）**：编辑器是用户参与入口，必须打磨到"零基础可上手"再外发
> 2. **V0.5 启动社区内容吸收**：把V0.2收集到的高质量用户地图精选进正式章节，减少内建内容压力
> 3. 新增模块（ConfigManager/ProgressFlags/LevelFlow/ChapterFlow/设置4Tab/工坊）全链路分散到对应迭代交付

### 🔹 V0.1 - 脚手架+配置骨架版 (Sprint 1, 周期: 1周)

**核心目标**：工程骨架 + 配置管理 + 进度Flags + 存档壳 + 基础输入物理；为V0.2编辑器交付打好基础

#### 依赖前置条件
- 引擎已安装（Godot 4.x，固定版本如4.2.1）
- 项目目录结构已创建

#### 任务清单 (Task List)

| TID | 任务 | 依赖 | 产出物 |
|-----|------|------|--------|
| T01-01 | 初始化项目、创建目录结构（含config/ data/ usermaps/） | 无 | 空项目工程文件 + 目录树 |
| T01-02 | 实现 GameManager 全局单例 + 单例注册脚本 | T01-01 | GameManager.gd + autoload.cfg |
| T01-03 | 实现 SceneManager 场景切换（淡入淡出） | T01-02 | SceneManager.gd |
| T01-04 | 实现 ResourceManager 资源加载 + JSON缓存 | T01-02 | ResourceManager.gd |
| **★T01-05** | **实现 ConfigManager（四层配置+dotPath+SetPref+Reload）** | T01-02 | ConfigManager.gd + 4个默认json模板 |
| **★T01-06** | **实现 ProgressFlags（字符串/KV + 序列化/反序列化）** | T01-02 | ProgressFlags.gd 单例 |
| T01-07 | 实现 InputManager 输入映射（A跳跃/ LT格挡/ Y冲刺/ X攻击） | T01-02 | InputManager.gd + input_map配置 |
| T01-08 | 配置碰撞矩阵（7层） + 单向平台规则 | T01-01 | 项目设置 Collision Layer/Mask |

| T01-10 | 创建测试场景 + 方块人占位角色 + 测试地面平台 | T01-08 | test_base.tscn |

#### 验收标准 (Definition of Done)
- [ ] 启动项目能看到测试场景，角色受重力下落落地停止
- [ ] 手柄（L摇杆/A/X/Y/LT）和键盘均有响应日志输出；LT按住有格挡状态
- [ ] **ConfigManager**：随机 20 次 dotPath 查询 0 错误；SetPref("audio.bgmVolume", 0.5) → 写盘 → 重启后读回一致
- [ ] **ProgressFlags**：Set("abc") + SetKV("k", 123) → Serialize → 新实例Deserialize → Has("abc") && GetKV("k")==123 成立
- [ ] **存档壳**：SaveSlot(1) → LoadSlot(1) → 地图/Flags 100% 还原
- [ ] 按Esc可退出；场景切换（主菜单→游戏场景）过渡平滑无报错

#### 风险与应对
| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| 手柄输入兼容 | 中 | 中 | 优先适配Xbox手柄；其余做fallback提示 |
| Config路径查询边界 | 低 | 高 | 缺失字段统一返回defVal+Warning日志，不崩溃 |
