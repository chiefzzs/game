## 三、Autoload 单例加载顺序（★关键：在 ProjectSettings → Autoload 中按序注册）

Godot Autoload 是全局单例，全局可通过 `ConfigManager.xxx` 直接调用。**加载顺序严格如下**（从上到下）：

| 序号 | 脚本路径 | 单例名（Node名） | 启用 | 说明（为什么这个顺序？） |
|------|---------|-----------------|------|-------------------------|
| 1 | `res://autoload/GameEvents.gd` | `GameEvents` | ✅ | 事件总线（最底层，所有人都可以发信号） |
| 2 | `res://autoload/ConfigManager.gd` | `Config` | ✅ | 配置（第二层，其他单例启动都要读配置） |
| 3 | `res://autoload/ProgressFlags.gd` | `Flags` | ✅ | 进度（依赖 Config 中的默认章节Flags） |
| 4 | `res://autoload/SaveSlotManager.gd` | `Saves` | ✅ | 存档（依赖 Flags 序列化 + Config 用户目录路径） |
| 5 | `res://autoload/InputBus.gd` | `Input` | ⚠️ 重命名冲突！→ `InputBus` | 输入总线（依赖 Config 的键位默认） |

> **⚠️ 坑点**：Godot 内置了 `Input` 单例，所以我们的输入总线重命名为 `InputBus`，Autoload 里 Node 名写 `InputBus`。

### 注册方法（Project→ProjectSettings→Autoload Tab）
```
逐个添加：
Path: res://autoload/GameEvents.gd → 点Add → Node Name自动GameEvents
Path: res://autoload/ConfigManager.gd → Node Name改为 Config（短！写代码方便）
...（其余依次）
最后，用上下箭头调整顺序为上表 1→2→3→4→5
```

### 4.1 ConfigManager（★V0.1 最核心，交付占比 50%）

#### 4.1.1 职责与四层配置加载策略
| 层级 | 枚举 | 加载路径 | 热加载 | 可写 | 说明 |
|------|------|---------|--------|------|------|
| L1 引擎常量 | `L1_CONSTANTS` | `res://config/L1_constants/constants.json` | ❌ 否 | ❌ 开发写 | 物理/碰撞/状态硬编码阈值，启动读一次后cache永不变 |
| L2 游戏平衡 | `L2_BALANCE` | `res://config/L2_balance/*.json`（多文件合并为一棵树） | ✅ 开发模式支持 | ❌ 策划写 | 角色/武器/敌人/掉落数值，开发时改了可一键重载不重启 |
| L3 关卡配置 | `L3_LEVELS` | `res://config/L3_levels/chapters.json` + 单地图 | ✅ 支持 | ⚠️ 策划/编辑器写 | 章节链路/地图目标/触发器，V0.1先占位chapters.json |
| L4 玩家设置 | `L4_USER` | `user://settings.json`（Godot user目录，Windows下 `%APPDATA%\Godot\app_userdata\MedievalRebellion\`） | ✅ 实时生效 | ✅ 玩家写 | 难度/音量/键位/画面，玩家改完 SetPref 立即存盘+发信号 |

> **四层合并查询规则（优先级从高到低覆盖）**：  
> 当调用 `Config.Get("x.y")` 时，先查 L4 用户设置 → 若缺省查 L3 → L2 → L1 → 返回 defVal。  
> **但为了V0.1简单清晰，我们不做跨层覆盖**，只在同层内查；改为加前缀：`Config.GetL1("...")` / `Config.GetL2("...")` / `Config.GetPref("...")`，这样更明确，也符合原需求文档里的 API。

#### 4.1.2 ConfigManager 类设计（完整 API + 字段）
```gdscript
# res://autoload/ConfigManager.gd
extends Node
## 全局配置管理器（Autoload单例，Node名：Config）
## 用法：
##   var jump : float = Config.GetL2("player.jumpForce", -500.0)
##   var vol : float = Config.GetPref("audio.bgmVolume", 0.8)
##   Config.SetPref("audio.bgmVolume", 0.5) # 自动存盘+发信号
##   Config.ReloadBalance() # 开发模式热加载L2

# ------------------------------------------------------------------
# 内部缓存（Dictionary 树结构，支持嵌套查询）
# ------------------------------------------------------------------
var _l1_cache : Dictionary = {}
var _l2_cache : Dictionary = {}   # 合并 player.json + difficulty.json + ...
var _l3_cache : Dictionary = {}
var _l4_cache : Dictionary = {}   # 运行中的玩家设置，改了立即写盘

const L1_PATH := "res://config/L1_constants/constants.json"
const L2_DIR := "res://config/L2_balance/"
const L3_CHAPTERS_PATH := "res://config/L3_levels/chapters.json"
const L4_DEFAULT_PATH := "res://config/L4_settings/settings.default.json"
const L4_USER_PATH := "user://settings.json"

# ------------------------------------------------------------------
# 信号（★所有人监听这些信号实现"配置变更响应"，比如音量改变立即调AudioServer）
# ------------------------------------------------------------------
signal PlayerPrefChanged(key: String, new_value)  # L4玩家设置改变
signal BalanceReloaded()                          # L2策划热改完成

# ================================================================
# 生命周期
# ================================================================
func _ready() -> void:
    _LoadL1()
    _LoadL2()
    _LoadL3()
    _LoadOrInitL4()

# ================================================================
# 对外通用查询接口（简化版：只查L2平衡配置+L4玩家设置，够用了）
# ================================================================

## 查询 L2 平衡配置（玩家/敌人/武器/难度等数值）
## path: 点分路径，如 "player.maxHp"；defVal: 默认值（类型驱动）
func GetL2(path: String, def_val = null):
    return _QueryDict(_l2_cache, path, def_val)

## 查询 L1 引擎常量（碰撞层/物理阈值等）
func GetL1(path: String, def_val = null):
    return _QueryDict(_l1_cache, path, def_val)

## 查询 L3 章节配置
func GetL3(path: String, def_val = null):
    return _QueryDict(_l3_cache, path, def_val)

## 【L4玩家设置】读取（带类型安全：def_val 类型不匹配时自动警告）
func GetPref(key: String, def_val = null):
    return _QueryDict(_l4_cache, key, def_val)

## 【L4玩家设置】写入 + 立即存盘 + 发 PlayerPrefChanged 信号
## 关键：V0.6 设置菜单实时生效的基础
func SetPref(key: String, value) -> void:
    _WriteDict(_l4_cache, key, value)
    _SaveL4()
    PlayerPrefChanged.emit(key, value)

## 把 L4 玩家设置全部重置为默认值（设置菜单「恢复默认」按钮用）
func ResetPrefsToDefault() -> void:
    _l4_cache = _LoadJson(L4_DEFAULT_PATH)
    _SaveL4()
    PlayerPrefChanged.emit("__ALL__", null)

## 开发模式：热重载 L2（编辑器脚本或控制台调用，避免每次重启）
func ReloadBalance() -> void:
    if OS.has_feature("editor") or OS.is_debug_build():
        _LoadL2()
        BalanceReloaded.emit()
        print("[Config] L2 balance reloaded successfully.")

## 【调试用】打印全部 L4 设置到控制台（V0.1验收测试用）
func DumpAllPrefs() -> void:
    print("=== L4 Player Settings ===")
    print(JSON.stringify(_l4_cache, "\t"))

# ================================================================
# 内部：JSON 加载/保存
# ================================================================
func _LoadL1() -> void:
    _l1_cache = _LoadJson(L1_PATH)
func _LoadL2() -> void:
    # 扫目录合并所有 json：player.json → {"player": {...}}, difficulty.json → {"difficulty": {...}}
    _l2_cache.clear()
    var dir := DirAccess.open(L2_DIR)
    if dir:
        dir.list_dir_begin()
        var fname := dir.get_next()
        while fname != "":
            if fname.ends_with(".json"):
                var key_name := fname.get_basename() # "player"
                var data := _LoadJson(L2_DIR + fname)
                _l2_cache[key_name] = data
            fname = dir.get_next()
func _LoadL3() -> void:
    _l3_cache = _LoadJson(L3_CHAPTERS_PATH)
func _LoadOrInitL4() -> void:
    if ResourceLoader.exists(L4_USER_PATH) or FileAccess.file_exists(L4_USER_PATH):
        _l4_cache = _LoadJson(L4_USER_PATH)
    else:
        _l4_cache = _LoadJson(L4_DEFAULT_PATH)
        _SaveL4()
func _SaveL4() -> void:
    _SaveJson(L4_USER_PATH, _l4_cache)

static func _LoadJson(path: String) -> Dictionary:
    var f := FileAccess.open(path, FileAccess.READ)
    if f == null:
        push_error("[Config] 无法加载: " + path + " 错误: " + str(FileAccess.get_open_error()))
        return {}
    var txt := f.get_as_text()
    f.close()
    var parsed = JSON.parse_string(txt)
    if typeof(parsed) != TYPE_DICTIONARY:
        push_warning("[Config] JSON 不是字典，返回空: " + path)
        return {}
    return parsed

static func _SaveJson(path: String, data: Dictionary) -> void:
    var f := FileAccess.open(path, FileAccess.WRITE)
    if f == null:
        push_error("[Config] 无法写入: " + path)
        return
    f.store_string(JSON.stringify(data, "\t"))
    f.close()

# ================================================================
# 内部：dotPath 查询/写入（核心算法）
# ================================================================

## 在 dict 树上按 "a.b.c" 查值，找不到返回 def_val
static func _QueryDict(dict: Dictionary, dot_path: String, def_val):
    if dict.is_empty():
        return def_val
    var parts := dot_path.split(".", false)
    var cur: Variant = dict
    for p in parts:
        if typeof(cur) == TYPE_DICTIONARY and (cur as Dictionary).has(p):
            cur = (cur as Dictionary)[p]
        else:
            return def_val
    return cur

## 在 dict 树上按 "a.b.c" 写值，中间字典不存在则自动创建
static func _WriteDict(dict: Dictionary, dot_path: String, value) -> void:
    var parts := dot_path.split(".", false)
    var cur: Dictionary = dict
    for i in range(parts.size() - 1):
        var p := parts[i]
        if not cur.has(p) or typeof(cur[p]) != TYPE_DICTIONARY:
            cur[p] = {}
        cur = cur[p]
    cur[parts[parts.size() - 1]] = value
```

### 4.2 ProgressFlags（进度标记系统，★ V0.1 核心 2）

#### 4.2.1 设计理念
游戏进度**不要用一堆散变量存**，统一用 2 个字典：
- `_flags`：`Dictionary<String, bool>` —— 简单布尔进度（已触发检查点cp1、已招募张三、已打开宝箱x…）
- `_kv`：`Dictionary<String, Variant>` —— 泛型键值（已得金币数、当前章节id、角色等级、存档时间戳…）

**所有写操作都必须通过 `Set()` / `SetKV()` 走，触发信号**，后续章节流程、UI、成就都监听这些信号。

#### 4.2.2 ProgressFlags 完整类
```gdscript
# res://autoload/ProgressFlags.gd
extends Node
## 全局进度标记（Autoload单例，Node名：Flags）
## 用法：
##   Flags.Set("cp_farm_01_01", true)        # 触发检查点
##   if Flags.Has("ch_prologue_done"): ...
##   Flags.SetKV("gold", 500)                # 存泛型
##   var g = Flags.GetKV("gold", 0)

var _flags: Dictionary = {}  # String -> bool
var _kv: Dictionary = {}     # String -> Variant (int/float/String/etc)

# ------------------------------------------------------------------
# 信号（其他人监听）
# ------------------------------------------------------------------
signal FlagSet(key: String, value: bool)
signal FlagCleared(key: String)
signal KVSet(key: String, value)
signal FlagsReset()  # 新游戏开始时，清空所有

# ================================================================
# API：布尔Flags（用得最多）
# ================================================================
func Set(key: String, value: bool = true) -> void:
    if key == null or key.is_empty():
        return
    var old = _flags.get(key, false)
    _flags[key] = value
    if old != value:
        FlagSet.emit(key, value)

func Unset(key: String) -> void:
    if _flags.has(key):
        _flags.erase(key)
        FlagCleared.emit(key)

func Has(key: String) -> bool:
    return _flags.get(key, false)

# ================================================================
# API：泛型KV
# ================================================================
func SetKV(key: String, value) -> void:
    if key == null or key.is_empty():
        return
    _kv[key] = value
    KVSet.emit(key, value)

func GetKV(key: String, def_val = null):
    return _kv.get(key, def_val)

func HasKV(key: String) -> bool:
    return _kv.has(key)

# ================================================================
# 新游戏：清空（Bootstrap 新建游戏时调）
# ================================================================
func ResetAll() -> void:
    _flags.clear()
    _kv.clear()
    FlagsReset.emit()

# ================================================================
# 序列化/反序列化（给 SaveSlotManager 存读档用）
# ================================================================
func ToDictionary() -> Dictionary:
    return {
        "schema_ver": 1,
        "flags": _flags.duplicate(true),
        "kv": _kv.duplicate(true),
        "saved_at_unix": Time.get_unix_time_from_system()
    }

func FromDictionary(data: Dictionary) -> bool:
    if not data.has("schema_ver") or data["schema_ver"] != 1:
        push_warning("[Flags] schema version mismatch, skip load.")
        return false
    ResetAll()
    if data.has("flags") and typeof(data["flags"]) == TYPE_DICTIONARY:
        _flags = (data["flags"] as Dictionary).duplicate(true)
    if data.has("kv") and typeof(data["kv"]) == TYPE_DICTIONARY:
        _kv = (data["kv"] as Dictionary).duplicate(true)
    # 恢复后，不发历史信号（避免读档触发一堆不必要的重放）
    return true

# ================================================================
# 调试
# ================================================================
func Dump() -> void:
    print("=== Progress Flags Dump ===")
    print("Flags(size=", _flags.size(), "): ", JSON.stringify(_flags))
    print("KV(size=", _kv.size(), "): ", JSON.stringify(_kv))
```

extends RefCounted

#### 4.3.3 SaveSlotManager 完整类
```gdscript
# res://autoload/SaveSlotManager.gd
extends Node
## 存档槽管理器（Autoload单例，Node名：Saves）
## 用法：
##   var slots_info = Saves.ListSlots()     # UI读：3个槽的摘要
##   Saves.NewGame(0)                       # 槽1新建（ProgressFlags清空）
##   Saves.Save(0)                          # 存到槽1
##   var ok = Saves.Load(0)                 # 从槽1读
##   Saves.Delete(0)                        # 删槽1

const SLOT_COUNT := SaveGameDTO.SLOT_COUNT

signal SlotSaved(slot_idx: int)
signal SlotLoaded(slot_idx: int)
signal SlotDeleted(slot_idx: int)

func _ready() -> void:
    # 确保 user://saves 目录存在
    if not DirAccess.dir_exists_absolute("user://saves"):
        DirAccess.make_dir_absolute("user://saves")

# ================================================================
# 列表：返回 3 个槽的摘要（存档菜单用）
# ================================================================
func ListSlots() -> Array[Dictionary]:
    var result: Array[Dictionary] = []
    for i in SLOT_COUNT:
        var path = SaveGameDTO.PathOf(i)
        if FileAccess.file_exists(path):
            var data = _ReadFile(path)
            if data.has("meta"):
                result.append({
                    "slot_idx": i,
                    "exists": true,
                    "last_played_at_unix": data.get("last_played_at_unix", 0),
                    "play_seconds_total": data.get("play_seconds_total", 0),
                    "meta": data.get("meta", {})
                })
                continue
        result.append({"slot_idx": i, "exists": false})  # 空槽
    return result

# ================================================================
# 新游戏：重置 ProgressFlags 为初始状态 + 写一个空的存档头
# ================================================================
func NewGame(slot_idx: int) -> void:
    Flags.ResetAll()
    # 写入初始flags（如果有默认的，V0.5章节链路再加）
    var dto := SaveGameDTO.MakeEmpty(slot_idx)
    dto.progress_flags = Flags.ToDictionary()
    _WriteFile(SaveGameDTO.PathOf(slot_idx), dto)
    SlotSaved.emit(slot_idx)

# ================================================================
# 保存：把当前ProgressFlags写入指定槽
# （V0.3以后扩展：写入Player/同伴快照，这里留好字段兼容）
# ================================================================
func Save(slot_idx: int) -> void:
    var path := SaveGameDTO.PathOf(slot_idx)
    var data: Dictionary
    if FileAccess.file_exists(path):
        data = _ReadFile(path)
    else:
        data = SaveGameDTO.MakeEmpty(slot_idx)
    # 刷新公共字段
    data["schema_ver"] = SaveGameDTO.SCHEMA_VERSION
    data["slot_idx"] = slot_idx
    data["last_played_at_unix"] = Time.get_unix_time_from_system()
    data["progress_flags"] = Flags.ToDictionary()
    # meta：V0.1 先给默认
    if not data.has("meta"):
        data["meta"] = {}
    _WriteFile(path, data)
    SlotSaved.emit(slot_idx)

# ================================================================
# 加载：把存档内容恢复 → 调 Flags.FromDictionary()
# ================================================================
func Load(slot_idx: int) -> bool:
    var path := SaveGameDTO.PathOf(slot_idx)
    if not FileAccess.file_exists(path):
        push_warning("[Saves] 槽 %d 没有存档" % slot_idx)
        return false
    var data := _ReadFile(path)
    # 1. 先恢复 Flags（最核心）
    if data.has("progress_flags"):
        var ok_flags = Flags.FromDictionary(data.progress_flags)
        if not ok_flags:
            push_warning("[Saves] 槽 %d Flags恢复失败" % slot_idx)
    # 2. V0.3以后：恢复 Player/Companions（这里先占位）
    SlotLoaded.emit(slot_idx)
    return true

# ================================================================
# 删除
# ================================================================
func Delete(slot_idx: int) -> void:
    var path := SaveGameDTO.PathOf(slot_idx)
    if FileAccess.file_exists(path):
        DirAccess.remove_absolute(path)
        SlotDeleted.emit(slot_idx)

# ================================================================
# 内部：JSON读写（V0.6后可加AES加密防玩家改，先不用）
# ================================================================
static func _ReadFile(path: String) -> Dictionary:
    var f := FileAccess.open(path, FileAccess.READ)
    if f == null:
        return {}
    var txt := f.get_as_text()
    f.close()
    var parsed = JSON.parse_string(txt)
    if typeof(parsed) == TYPE_DICTIONARY:
        return parsed
    return {}

static func _WriteFile(path: String, data: Dictionary) -> void:
    var dir := path.get_base_dir()
    if not DirAccess.dir_exists_absolute(dir):
        DirAccess.make_dir_absolute(dir)
    var f := FileAccess.open(path, FileAccess.WRITE)
    if f == null:
        push_error("[Saves] 写失败: " + path)
        return
    f.store_string(JSON.stringify(data, "\t"))
    f.close()
```

### 4.5 InputBus（输入总线，★ V0.1 输入骨架）
> **设计思想**：把「物理按键」和「动作语义」解耦。  
> 例如：玩家改键后 L3 按下 = 冲刺，我们其他代码不用改，全监听 `InputBus.DashPressed()` 这个信号。

#### 4.5.1 第一步：在 ProjectSettings → InputMap 里定义这些动作（V0.1必须建）

| 动作名（Action） | 键盘按键 | 手柄按键（Xbox） | 说明 |
|-----------------|---------|------------------|------|
| **move_left** | A | L 左推 / DPad 左 | 移动左 |
| **move_right** | D | L 右推 / DPad 右 | 移动右 |
| **jump** | Space / W | A | 跳跃 |
| **attack** | J | X | 攻击（X键） |
| **dash** | K | Y | 冲刺（Y键） |
| **block** | 左Shift | Left Trigger（LT，轴0~1） | **格挡（用户要求的LT键）** |
| **interact** | E | B | 互动/对话/拾取 |
| **pause** | Esc | Start | 暂停游戏 |
| **ui_cancel** | Esc | B | UI返回（复用） |
| **ui_accept** | Enter / Space | A | UI确认 |

> **LT 轴判断要点**：Godot 中 LT 默认是轴（-1~1），需要在 InputMap 里把「block」动作的手柄项设为 Joypad Button Axis 并设阈值 ≥ 0.35（V0.1 先这么写，V0.6 设置菜单改键时再细化）。

#### 4.5.2 InputBus 类
```gdscript
# res://autoload/InputBus.gd
extends Node
## 输入总线（Autoload单例，Node名：InputBus，避免和Godot内置Input重名）
## 其他代码监听以下信号，不要直接 _input() 里读按键

# ----动作信号：Pressed=按下瞬间；Held=每帧；Released=松开----
signal MoveLeftPressed()
signal MoveRightPressed()
signal JumpPressed()
signal JumpReleased()
signal AttackPressed()
signal DashPressed()
signal BlockPressed()
signal BlockReleased()
signal InteractPressed()
signal PausePressed()

## 每帧可轮询的强度（0~1），比如格挡按多少LT
var blockStrength: float = 0.0  # 0 没按 → 1 按到底
var moveAxis: float = 0.0       # -1 左 → 1 右

func _process(delta: float) -> void:
    # ----1. 先算持续的Held值----
    moveAxis = Input.get_axis("move_left", "move_right")
    blockStrength = clamp(Input.get_action_strength("block"), 0.0, 1.0)

# ----2. 离散的Pressed/Released事件（用_is_action_just_pressed）----
func _unhandled_input(event: InputEvent) -> void:
    if event.is_action_pressed("jump"):     JumpPressed.emit()
    if event.is_action_released("jump"):    JumpReleased.emit()
    if event.is_action_pressed("attack"):   AttackPressed.emit()
    if event.is_action_pressed("dash"):     DashPressed.emit()
    if event.is_action_pressed("block"):    BlockPressed.emit()
    if event.is_action_released("block"):   BlockReleased.emit()
    if event.is_action_pressed("interact"): InteractPressed.emit()
    if event.is_action_pressed("pause"):    PausePressed.emit()
    if event.is_action_pressed("move_left"): MoveLeftPressed.emit()
    if event.is_action_pressed("move_right"): MoveRightPressed.emit()

## 工具：是否正按住跳跃（可变高跳用）
func IsJumpHeld() -> bool: return Input.is_action_pressed("jump")
## 工具：是否正按住格挡
func IsBlockHeld() -> bool: return blockStrength > 0.35
```

## 八、技术风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| Godot 4.6.2 没有 Forward+ 设备（老核显） | 中 | 低 | project.godot 渲染器改成 GL Compatibility，一行的事 |

| SaveSlotManager 用户目录权限写不了 | 低 | 高 | 加 `DirAccess.make_dir_absolute` + 写失败时弹窗「请用管理员运行/检查杀毒」 |
| Autoload 顺序错了，导致启动报错 Config 未定义 | 高 | 中 | 文档里写死 Autoload 顺序 + 代码 Bootstrap 里加断言，不对就主动报错 |
