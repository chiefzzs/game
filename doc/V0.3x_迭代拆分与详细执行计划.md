### V0.3b — CombatDamageCalculator (CDC) 七步流水线

**交付目标**：纯逻辑模块，独立于角色系统。给定 attacker/victim Dictionary + 参数 → 返回结构化战斗结果（满足文档 §2.3 字段）。

**输入（前置）**：
- V0.3a：5 套 JSON 存在且可读取；DamageType 枚举已定义。

**输出（2 个文件）**：

| # | 文件路径 | 内容要点 |
|---|----------|----------|
| B1 | `scripts/combat/CombatDamageCalculator.gd` | `extends RefCounted`, `class_name CombatDamageCalculator`；静态方法 `calculate(attacker: Node, victim: Node, raw_atk: int, dtype: String, dir: Vector2, shield_break: bool, atk_mult: float) -> Dictionary`；严格按文档 §2.1 7步顺序：①防禦相減 → ②方向判定 → ③格挡 → ④背刺 → ⑤暴击 → ⑥min/max clamp → ⑦击退；每步 append steps[] 字符串便于调试 |
| B2 | `scripts/test/CombatDamageSmokeTest.gd` | `extends RefCounted`；`run() -> (pass_cnt, fail_cnt, Array[String] logs)`；覆盖 §2.2 UC1~UC10，每个用例用 Dictionary 构造 mock attacker/victim（不必是 Node） |

**验收标准**：
- Headless runner `runner_combat_damage.gd` 执行 CombatDamageSmokeTest：UC1~UC10 **10/10 通过**；
- 步骤 log 中 `steps.size()` 恒等于 7（每步都打了 trace）；
- UC9（体力不足格挡）验证：stamina < cost 时，final_damage 虽减但仍 > 0。

**风险**：
- 背刺判定方向 dir：需和 CharacterBase.facing (±1) 对齐全局约定；
- shield_break 武器优先于 block 状态（UC10）顺序必须在第③步之前判断。

**输入（前置）**：V0.3a（枚举）、V0.3b（伤害调用）

**输出（2 个文件）**：

|---|----------|----------------|
| C1 | `scripts/editor/CharacterBase.gd` | `extends CharacterBody2D`, `class_name CharacterBase`；`_autoload(name)` 安全取单例函数；**字段**：kind/state/prev_state/id_key/display_name/facing、hp/max_hp、stamina/max_stamina、base_atk/defense、is_invincible、hp_changed/stats_changed/hit/died 信号；**方法**：`take_damage(res)`（调用后进入 HURT 状态、hp 扣减、信号）、`heal(n)`、`die()`（切 DEAD、关碰撞、延迟 queue_free）、`change_state(to)` 前置检查（§3.1 状态跳转表）、每帧 `recover_stamina(dt)` |
| C2 | `scripts/test/FSMBasicSmokeTest.gd` | 8 条状态转换用例：①IDLE→MOVE 合法；②IDLE→ATTACK 合法；③IDLE→JUMP 合法；④ATTACK→MOVE 不合法（留在 ATTACK）；⑤DEAD 不可切任何；⑥take_damage 切 HURT；⑦HP 归 0 自动切 DEAD；⑧BLOCK 中发伤害触发 block 减伤 |

**验收标准（Headless 8/8 PASS）**：
- FSMBasicSmokeTest.run() → pass=8 fail=0；

**风险**：
- `extends` / `class_name` 行必须写在文件最顶部（任何函数之前），否则 Could not resolve super class。

## 3. 关键防坑规则（总结历史踩坑）

> 来自 Terminal#1-1030 及多次 revert 的经验教训。每个 V0.3x 验收前**必须自查**。

### 3.1 GDScript 语法 / 注册

| 坑 | 规避方法 |
|----|----------|
| "Could not resolve super class inheritance from PlayerBase" | `extends "res://path.gd"`（必须是字符串）；extends/class_name 必须在非注释第一行，前面不允许有 func/const 定义（除注释） |
| "Too many arguments for 'get()' call" | Godot 4 `Node.get("x", 123)` 非法 → 统一 `val = n.get("x") if n.has("x") else 123`；Dictionary.get(key, default) 同理 |
| "Identifier not declared: CharacterEnums" | ① CharacterEnums.gd **不得写 class_name**；② 所有引用它的脚本顶部加 `const _CE := preload("res://scripts/config/CharacterEnums.gd")`；③ 用 `_CE.BaseState.IDLE`；④ 子类自动继承父类 `_CE`，**不要重复定义** |
| "The member '_CE' already exists in parent class" | 子类不要再定义一遍同名 const；replace_all 时**不能把 preload 字符串内的路径也替换**（例如 `"res://..._CE.gd"` 是错误） |
| Parse Error: Unexpected "class_name" | `class_name` 必须在 extends 之后、任何 const/func/var 之前；注释允许 |

### 3.2 Headless 测试

| 坑 | 规避方法 |
|----|----------|
| "Cannot infer the type of 'validator' variable" | Godot 4.6.2 严格模式：所有 `:=` 从 load().new() 推断失败时，显式写类型：`var validator_script: Script = load(path); var validator: RefCounted = validator_script.new()` |
| `add_child(parent)` SceneTree 中找不到 | `extends SceneTree` 的无头脚本中，要写 `root.add_child(...)` 而不是直接 `add_child` |
| Engine.set_return_code 不存在 | 老代码兼容：改用 `quit(code)`（SceneTree.quit() 带参数）即可 |
| Headless 中 Config 单例取不到 | 必须先确保 ConfigManager 在 autoload 注册，且 Headless runner _init 中先尝试 `get_tree().root.get_node_or_null("/root/Config")` |

### 3.3 CMD / Windows 编码

| 坑 | 规避方法 |
|----|----------|
| "系统找不到指定的路径" | .cmd 使用**相对路径**和 `pushd "%~dp0..\.."` 回项目根；Godot 路径用绝对路径并 if exist 判断 |
| 中文 / 进度条乱码 | .cmd 保存为 **GBK + CRLF**；echo 中文前 `chcp 936 >nul` |
| PowerShell 执行策略拒绝 .ps1 | 项目只交付 .cmd，用 Godot.exe 原生 .exe 调用不走 .ps1 |

### 3.4 Git & 版本恢复

| 坑 | 规避方法 |
|----|----------|
| 中文文件名 `git checkout` 失败 | 用通配符路径恢复（`git checkout HEAD~1 -- doc/ cmd/v0.2/`），不要列出每个中文文件 |
| 想回滚代码但保留 doc | 先把 doc 目录复制到项目外临时目录 → git checkout target -- . → 再覆盖回 doc |

| A06 | **Node 引用 / `get_node()` / `$路径` 必须放在 `_ready()` 之后执行**；严禁在脚本顶部 `var hp_bar: ProgressBar = $UI/HpBar`（初始化时场景树还没构建，直接 nil） | 经典 Bug：顶部写 `var target = $Player` → 运行时报空指针，但 stack trace 指向构造函数，不知道为什么 | ✅ `@onready var target: CharacterBase = $Player`<br>（Godot 4 用 `@onready`，等价于在 _ready 第一行赋值） |

### 7.4 Godot 4.6 运行时常见坑的防御性编程（90% 崩溃都在下面）

| 编号 | 铁律 | 违反后果（历史案例） | 防御代码模板 |
|------|------|-------------------|-------------|
| D01 | **所有 `connect()` 信号必须用 `is_connected()` 判重**；尤其是 Autoload 单例在热重载下会 connect 多次，导致一次 emit 回调 N 次（连击数翻倍、金币翻倍） | V0.2 GameEvents.level_started 每次热重载多连一次 → 跑 3 次测试后关卡开始事件触发 4 次，玩家被生成了 4 个 | ```gdscript

    var ge := get_tree().root.get_node_or_null("/root/GameEvents")
    if ge and not ge.enemy_died.is_connected(_on_enemy_died):
        ge.enemy_died.connect(_on_enemy_died)

| D02 | **所有跨节点引用、Autoload 引用、`get_node()` 结果，使用前必须判 `is_instance_valid()` 或 null**；特别是异步回调 / 信号延迟回来时节点可能已经被 `queue_free()` 了 | V0.3 敌人 AI 里 `target.queue_free()` 后 1 帧信号才到 → 「Attempt to call function 'get_global_position' in base 'previously freed instance'」，直接崩 | ```gdscript

``` |
| D03 | **`CharacterBody2D.move_and_slide()` 必须在 `_physics_process(delta)` 里调用**，严禁写在 `_process` / `_input` / 信号回调里 | V0.2 某玩家脚本把 move 写在 `_input` 里 → 60Hz 屏幕正常，144Hz 下人物速度快 2.4 倍，敌人 AI 永远追不上 | ```gdscript
func _physics_process(delta: float) -> void:
    velocity = input_dir * SPEED
    move_and_slide()  # 只有这里能调用
``` |
| D04 | **Tween / AnimationPlayer 播放前必须判节点存活**，且建议用 `create_tween().bind_node(self)` 模式；节点被 free 时 tween 自动停止，不会继续跑回调炸空指针 | V0.2 敌人死亡动画播到一半被 queue_free → Tween 回调还在 `queue_redraw()`，炸 Invalid set index | ```gdscript

| D06 | **`queue_free()` 之后严禁再访问该节点任何属性 / 方法**；如果需要发死亡事件，**必须先 emit 信号再 queue_free**，顺序不能反 | V0.3 早期 EnemyBase `die()` 先写 `queue_free()` 再 `GameEvents.enemy_died.emit(self, ...)` → self 已经被标记删除，emit 的参数 who 是一个 invalid instance，PickupSystem 收到后访问 global_position 直接炸 | ✅ 正确顺序（死亡铁三角）：<br>`func die():`<br>&nbsp;&nbsp;&nbsp;&nbsp;`state = _CE.BaseState.DEAD`<br>&nbsp;&nbsp;&nbsp;&nbsp;`GameEvents.enemy_died.emit(self, id_key, global_position)`  # **1. 先发信号（此时 self 还活着）**<br>&nbsp;&nbsp;&nbsp;&nbsp;`set_physics_process(false)`  # **2. 停物理，防止死亡后还继续移动**<br>&nbsp;&nbsp;&nbsp;&nbsp;`queue_free()`  # **3. 最后再销毁** |

### 7.6 代码修改前的最后 5 条自检清单（提交代码前默念一遍，能防 80% 的 Bug）
