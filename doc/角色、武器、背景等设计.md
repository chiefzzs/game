2D 像素游戏角色图元美术规范文档（适配 Godot 4.6.2）
一、项目基础规范
1. 美术风格基准
沿用初始橙色圆角方块像素角色风格：
像素风、1px 纯色黑色轮廓描边，色块平涂无渐变；
2D 侧视视角（左右双向动作）；
分层设计：无武器基础角色层 + 独立武器 / 道具图层，图层分离便于武器换装；
画布透明背景 PNG 格式，导入 Godot 时纹理过滤设置为「Nearest（邻近采样）」，避免模糊。
2. 两种标准图元尺寸（双规格配套）
表格
画布尺寸	使用场景	画布填充规则
32×32 px	主角、NPC、士兵、骑士、领主等大型单位	角色主体占 24~30px，上下左右预留 2~4px 安全边，无多余空白
16×16 px	小怪、场景装饰、粒子、小型生物（萤火虫、蘑菇、荷叶平台）	主体占 12~14px，紧凑贴合画布，减少留白
3. 动作帧统一标准（所有人形单位共用 6 套动作）
全部分为朝左、朝右两套镜像动作，每套包含 6 帧：
Idle_R：朝右静止站立（基础待机）
Idle_L：朝左静止站立
Dash_R：向右冲刺移动
Dash_L：向左冲刺移动
Jump_Up：上升跳跃帧
Jump_Down：下落跳跃帧
二、分层美术设计方案（核心换装逻辑）
1. 分层结构（Godot 节点对应）
plaintext
CharacterBody2D（角色根节点）
├─ Sprite2D_Body  # 图层1：无武器基础角色本体（核心层）
├─ Sprite2D_Weapon # 图层2：独立武器/盾牌叠加层（上层，覆盖手部）
├─ CollisionShape2D  # 碰撞盒：人形单位统一24×28(32px画布)/12×14(16px画布)
└─ AnimationPlayer  # 驱动6套动作帧切换
2. 图层制作要求
基础角色层（Body）
所有角色手部握持位置留白，不绘制任何武器、工具、盾牌；
全角色统一右手锚点坐标（32px 画布：(12,22)；16px 画布：(6,11)），保证所有武器贴合位置一致；
仅绘制躯干、头部、四肢基础轮廓，适配所有武器换装。
独立武器道具层（Weapon）
武器单独一张透明 PNG，不含身体；
武器握柄锚点与角色手部锚点完全对齐，无需手动偏移；
盾牌区分前后层级：手持小盾同层，重甲大盾可设置在身体后方子图层。

四、Godot 导入 & 使用配置规范
纹理导入设置
压缩：VRAM 无损压缩；
过滤模式：Nearest；
开启透明通道；
取消自动缩放，保留原始像素尺寸。
动画制作规范
AnimationPlayer 中分开存储身体动画轨道、武器显示轨道；
换装逻辑代码示例：
gdscript
@onready var body_sprite: Sprite2D = $Sprite2D_Body
@onready var weapon_sprite: Sprite2D = $Sprite2D_Weapon
# 切换武器函数
func set_weapon(weapon_path: String):
    weapon_sprite.texture = load("res://assets/sprites/weapons/" + weapon_path + ".png")
锚点统一规则
所有角色 Sprite 锚点设置：底部中心，适配跳跃、落地碰撞逻辑。
五、素材交付说明
素材分为两大文件夹：
/sprites/32x32：32 像素规格素材，内含body/角色本体、weapons/独立武器两个子目录；
/sprites/16x16：16 像素规格素材，目录结构与 32x32 完全一致；
每个角色文件夹内包含 6 动作帧（左右双向），场景物件单独归类；
所有图片为透明底 PNG，无白底冗余像素；
无法直接提供压缩包，文档内附美术生成标准描述，可按本规范批量生成全部图元素材后手动打包使用。
六、基准参考图说明
文档基准视觉原型：橙色圆角方块头部、双竖线简约像素眼睛，纯色平涂像素 2D 侧视造型，全角色统一这套简约像素美术语言，保证游戏画风统一。
