## 2. 子模块详细设计

### 2.1 FSM 11 状态跳转表（PlayerBase 专用，扩展自 CharacterBase 8 态）

| 源状态 | 事件 | 目标状态 | 条件 | 失败返回错误码 |
| --- | --- | --- | --- | --- |
| 任意非HURT/DEAD | `take_damage` 非致命 | HURT | is_dead=false | ERR_DOES_NOT_EXIST(33) |
| 任意非DEAD | `take_damage` HP<=0 | DEAD | - | ERR_DOES_NOT_EXIST(33) |
| IDLE/RUN | `_on_jump_pressed` 地面 | JUMP | jump_count==0 | ERR_DOES_NOT_EXIST |
| JUMP | `_on_jump_pressed` 空中 | DOUBLEJUMP | jump_count==1（<max=2） | ERR_DOES_NOT_EXIST |
| IDLE/RUN | `_on_attack` 窗口内第1/2/3次 | ATTACK1/2/3 | stamina≥5 + combo_lock=false | ERR_DOES_NOT_EXIST |
| IDLE/RUN | `_on_block_pressed` 长按 | BLOCK | stamina≥20 | ERR_DOES_NOT_EXIST |
| BLOCK | `_on_block_released` 松键 | IDLE | - | ERR_DOES_NOT_EXIST |
| IDLE/RUN | `_on_dash` LeftShift | DASH | stamina≥20 + dash_cd_left<=0 | ERR_DOES_NOT_EXIST |
| ATTACK× / HURT / DASH 结束 | 计时器到时 | IDLE | - | ERR_DOES_NOT_EXIST |
| DEAD | 任何输入 | DEAD（自锁） | is_dead=true（永远） | ERR_DOES_NOT_EXIST |

### 2.2 体力系统（Stamina）公式与数值

| 动作 | 体力消耗 | CD |
| --- | --- | --- |
| 平时不举盾不动作 | +15 / 秒（回复） | - |
| K 举盾 BLOCK | -10 / 秒（硬消耗） | - |
| 攻击 ATTACK 1/2/3 | -5 / 每次 | 连击窗口内不算额外CD |
| Shift 冲刺 DASH | -20 / 每次 | 0.8 秒内禁止再冲 |
| 切武器 1/2/3 | 0 | 无CD |

### 2.3 武器配置（FarmerPlayer 默认 3 武器，可 Config 覆盖）

```jsonc
{
  "fist": { "atk_mult": 1.0, "range": 36.0, "break_shield": false },   // 键1 默认
  "axe":  { "atk_mult": 1.3, "range": 46.0, "break_shield": true  },   // 键2 破盾
  "bow":  { "atk_mult": 0.9, "range": 160.0, "break_shield": false }   // 键3 远程
}
```
> 设计说明：break_shield=true 指被攻击对象若处于 BLOCK 状态，仍能造成伤害（忽略盾防）。斧头对训练Dummy举盾目标仍有效。

## 6. Godot 4.6 代码开发强制原则（追加在本设计文档最后，供后续 V0.3e~V0.3h 遵守）

1. ✅ **extends 必须首行**：任何 `.gd` 文件第一行必须是 `extends Xxx`，禁止前面有注释/空行
2. ✅ **全类型声明**：所有变量/参数/返回值必须加 `: Type`（`var hp: int = 100` / `func f(x: int) -> void`）
3. ✅ **class_name 禁止在 autoload 中使用**：单例请用 `SceneTree.root.AutoloadName` 或 `_autoload("Xxx")` 安全访问
4. ✅ **信号参数类型声明 + 连接判重**：`signal my_sig(a: int, b: String)`；连接前 `if not s.is_connected(cb): s.connect(cb)`
5. ✅ **assert 仅限测试脚本**：业务代码禁止 `assert`，使用 `push_error("...") + return`
6. ✅ **preload 仅在文件顶部常量**：禁止函数内部 `preload`；函数内用 `load(path)` 或 `ResourceLoader.load`
7. ✅ **路径大小写敏感**：`res://Scripts/X.gd` ≠ `res://scripts/x.gd`，永远小写蛇形 `scripts/characters/farmer_player.gd`（本项目遵循实际目录）
8. ✅ **信号连接参数数量&类型完全匹配**：InputBus 信号 0 参数 → 回调 0 参数；禁止 `_on_jump_pressed(_held)` 这种多余参数（V0.3d 修复的 bug）
9. ✅ **`move_and_slide` 只在 `_physics_process(delta)` 内调用**：禁止 `_ready`/`_process` 中移动角色
10. ✅ **Tween 绑定节点 + 节点释放自动停**：`var t = create_tween().bind_node(self)`；禁止全局 Tween 引用游离节点
11. ✅ **碰撞层用枚举禁止硬编码数字**：`collision_layer = 1 << CollisionLayer.GROUND`（V0.3c CharacterBase 已定义枚举）
12. ✅ **死亡处理顺序：`is_dead=true` → `change_state(DEAD)` → `died.emit()` → 延迟 `queue_free()`**：顺序错乱会导致回调者空指针
13. ✅ **无头测试 RefCounted + Dictionary 模拟**：不要继承 Node；`var p = PlayerBase.new(); p._ready();` 直接实例化测试
14. ✅ **GBK 编码 + CRLF 行尾 .cmd 脚本**：Windows .cmd 必须用 GBK 保存，UTF-8 会乱码 → `cmd /c chcp 936 >nul` 修复（V0.2 已统一）
15. ✅ **One Track 文件名加版本号后缀避免冲突**：V0.3x 场景/测试/脚本命名 `V03{d,e,f,g,h}_Xxx.{gd,tscn}`，永远不覆盖 V0.1/V0.2/V0.3a~c 同名文件
