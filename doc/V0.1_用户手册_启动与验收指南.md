## 5. 验收场景操作手册（QA 必读）

V0.1 共 4 个验收场景，其中 3 个**全自动**（进场景自动跑完，看 `Failed` 是否为 0 即可），1 个**手动**（按键盘/手柄验证输入与碰撞）。

### 5.1 验收 1：Config 四层查询（全自动）
- **进入方式**：主菜单点 🧪 Config测试 或 **F1**
- **耗时**：< 1 秒
- **用例数量**：20 项（C-01 ~ C-20）
- **通过标准**：屏幕底部应出现
  ```
  Total: passed=20 failed=0
  ```
- **覆盖内容**：L2 玩家数值/L2 难度/L1 碰撞&物理/L3 章节/L4 默认偏好/SetPref 持久化/PlayerPrefChanged 信号/ResetPrefs/L2 热重载

### 5.2 验收 2：ProgressFlags 进度标记（全自动）
- **进入方式**：主菜单点 🧪 Flags测试 或 **F2**
- **耗时**：< 1 秒
- **用例数量**：6 项（F-01 ~ F-05，其中 F-04 拆 2 个断言）
- **通过标准**：
  ```
  Passed 6 / Failed 0
  ```
- **覆盖内容**：Set/Unset/Has、KV 存取、ToDictionary→ResetAll→FromDictionary 还原、FlagSet 信号对「同值重复 Set」不重复触发

### 5.3 验收 3：SaveSlotManager 多槽存档（全自动）
- **进入方式**：主菜单点 🧪 Save测试 或 **F3**
- **耗时**：< 1 秒
- **用例数量**：11 项
- **通过标准**：
  ```
  Passed 11 / Failed 0
  ```
- **覆盖内容**：3 个槽固定尺寸、NewGame 产生槽、Save→ResetAll→Load 还原 flags & KV、**槽与槽互不污染**、Delete 清槽

### 5.4 验收 4：InputBus 输入 + 碰撞层（**手动**）
- **进入方式**：主菜单点 🧪 输入/碰撞 或 **F4**
- **预计操作时间**：1~2 分钟
- **场景内容**：一块地面（绿色大草地）+ 两个棕色浮空平台 + 一个橙色小农夫（占位角色块）
- **操作步骤与通过标准**（必须全部观察到）：

| # | 操作 | 现象（HUD 日志必须打出对应行） |
| --- | --- | --- |
| 1 | 按 `A` 键 | 角色向左走；HUD 出现 `➡️ MovePressed( dir=-1.0 )` + `↕️ moveAxis=-1.000` |
| 2 | 按 `D` 键 | 角色向右走；dir=+1.0 |
| 3 | 按 `W` 或 `↑` | `⬆️ MoveYPressed( up=+1.0 )`（UI/上爬/格挡姿态的语义轴，V0.1 先占位） |
| 4 | 按 `Space` 键（短按） | 角色跳离地面；HUD 出现 `🦘 JumpPressed( held=False )` + `🦘 JumpReleased`（若短按） |
| 5 | 按住 `Space` 1 秒再松 | 跳得更高；依次打印 `JumpPressed(True)` → 松手 → `JumpReleased` |
| 6 | 按 `J` 键 | `🗡️ AttackPressed( combo_step=... )` |
| 7 | 连按 `J` 三次 | `AttackPressed` 出现 3 次（V0.2 才真正实现 combo 伤害，这里只验证 InputBus 事件） |
| 8 | 按 `K` 键 | `💨 DashPressed` |
| 9 | 按住 `L` 键 2 秒再松 | HUD 连续打印 `🛡️ Block( strength=1.000 )` 多行，松后 Block strength 归 0 |
| 10 | **手柄** 推左摇杆左/右 | `moveAxis` 0.000→±1.000 过渡，中间有 0.2 死区 |
| 11 | 手柄按 **X** / **□** | `AttackPressed` |
| 12 | 手柄按 **A** / **✕** | `JumpPressed` |
| 13 | 手柄按 **RB / R1** | `DashPressed` |
| 14 | 手柄按住 **LT / L2**（模拟扳机） | `Block( strength=... )` 连续，strength 从 0 到 1 渐进；松开后归 0 |
| 15 | 手柄按 **Start** 键 | H 行出现 `⏸ InputBus.PausePressed 信号触发` |
| 16 | **碰撞**：跳上第 1 个平台（棕色） | 角色站在平台上，不会穿下去（CharacterBody2D collide_mask 只与 4 层地面碰撞） |
| 17 | 跳上第 2 个更高的平台 | 同上 |
| 18 | 从平台边缘掉回绿色地面 | 掉落后稳稳落在地面，不会穿出 |

> 通过标准：**以上 18 项全部能复现**，即为 InputBus + 碰撞层验收通过。

最后按 `Esc` 回主菜单。

## 6. 全局按键速查

| 语义 | 键盘 | 手柄（Xbox布局） | 对应 InputBus 信号 |
| --- | --- | --- | --- |
| 左右移动 | `A / D`，`← / →` | 左摇杆 X / 方向键 | `MovePressed(dir)`、`moveAxis` |
| 上下（UI/攀爬轴） | `W / S`，`↑ / ↓` | 左摇杆 Y | `MoveYPressed(up)`、`moveYAxis` |
| 跳跃 | `Space` | **A 键（✕）** | `JumpPressed(held)`、`JumpReleased` |
| 普通攻击 | `J` | **X 键（□）** | `AttackPressed(combo_step)` |
| 冲刺 | `K`、`ShiftL` | **RB（R1）** | `DashPressed` |
| 格挡（按压力度） | `L`（1/0） | **LT（L2）扳机 0~1** | `Block(strength)`（连续发射，strength 归一） |
| 交互/拾取（占位） | `E` | **Y 键（△）** | `InteractPressed` |
| 暂停/菜单 | `Esc`、`P` | **Start** | `PausePressed`、`PauseReleased` |
| （QA 便利）切测试场景 | `F1 / F2 / F3 / F4` | — | 切 Config/Flags/Save/输入碰撞 测试场景 |

> 所有 InputBus 信号的**发射端**集中在 `InputBus.gd`，业务脚本严禁直读 Input，全部走信号订阅。

## 8. 常见问题 FAQ

**Q1：启动后黑屏几秒后直接报错？**  
A：请确认：1）Godot 版本必须是 4.6.2；2）命令行用的是你本机的 exe 路径；3）启动时当前目录 `project.godot` 存在。

**Q2：主菜单按钮点「新游戏」后，settings.json / slot_01.sav 我找不到？**  
A：它们在 `user://` 下，真实位置是 `%APPDATA%\Godot\app_userdata\game1\`，直接在资源管理器路径栏粘贴这段即可跳转。

**Q3：手柄 LT 怎么按都只出 Block 1.0 或者 0？**  
A：LT 是扳机键（模拟量），要**慢慢扣**观察 strength 从 0.0→1.0 的连续行；如果你的手柄驱动把 LT 映射成数字键（无压力感应），则只会出现 0.0 和 1.0 两档，这也是符合预期的。

**Q4：F1~F4 在编辑器里切不进去？**  
A：编辑器模式下 F1~F4 是 Godot 自己的快捷键；**请按 F5 进入运行态**，在运行态窗口里按 F1~F4 才会切场景。

**Q5：想直接在编辑器里打开测试场景？**  
A：在「FileSystem」面板里打开 `scenes/test/*.tscn`，按 F6 可「运行当前场景」。

**Q6：git 首次 commit 完成了，我想加新的文件怎么提交？**
```powershell
git add 你改的文件路径
git commit -m "feat/xxx: 你干了什么"
```
